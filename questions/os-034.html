零拷贝技术

操作系统 困难
标签：零拷贝, sendfile, mmap, DMA, IO优化

📑 题目

什么是零拷贝技术？常见的实现方式有哪些？

💡 参考答案

零拷贝：减少或消除数据在内核空间和用户空间之间的拷贝次数，提高IO性能。

传统IO流程（4次拷贝）：
1. 磁盘→内核缓冲区（DMA拷贝）
2. 内核缓冲区→用户缓冲区（CPU拷贝）
3. 用户缓冲区→Socket缓冲区（CPU拷贝）
4. Socket缓冲区→网卡（DMA拷贝）

零拷贝技术：

1. mmap + write：
- 将文件映射到用户空间内存
- 减少一次CPU拷贝（共3次拷贝）
- 仍需CPU参与从内核到Socket缓冲区

2. sendfile：
- 在内核空间直接将数据从文件缓冲区传输到Socket
- 减少到2次拷贝（DMA+DMA）
- 带有scatter/gather的sendfile只需2次拷贝+1次描述符传递

3. splice：
- 在两个文件描述符之间移动数据
- 零拷贝，至少一个必须是管道

应用场景：
- 静态文件服务器（Nginx）
- Kafka消息传输
- 文件上传下载

💻 代码示例

// 传统方式：read + write（4次拷贝） char buf[8192]; int fd = open("file.txt", O_RDONLY); while ((n = read(fd, buf, sizeof(buf))) > 0) { write(socket_fd, buf, n); // 用户空间中转 } // mmap方式（3次拷贝） void* addr = mmap(NULL, file_size, PROT_READ, MAP_PRIVATE, fd, 0); write(socket_fd, addr, file_size); munmap(addr, file_size); // sendfile方式（2次拷贝，零拷贝） off_t offset = 0; sendfile(socket_fd, fd, &offset, file_size); // splice方式 int pipefd[2]; pipe(pipefd); splice(fd, &offset, pipefd[1], NULL, file_size, SPLICE_F_MOVE); splice(pipefd[0], NULL, socket_fd, NULL, file_size, SPLICE_F_MOVE); // Nginx配置零拷贝 // sendfile on;
