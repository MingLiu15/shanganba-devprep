元类（metaclass）的原理

Python 困难
标签：高级特性, 元类, metaclass, 类创建

📑 题目

解释Python元类的概念、工作原理及实际应用场景。

💡 参考答案

元类（metaclass）是创建类的类。普通类是创建对象的模板，元类是创建类的模板。type就是Python的默认元类。

工作原理：
- 定义类时，Python调用元类的__new__和__init__方法来创建类对象。
- 类的定义过程本质上就是「实例化元类」的过程。
- class MyClass(Base, metaclass=MyMeta) 等价于 MyClass = MyMeta('MyClass', (Base,), {...})。

元类的__new__方法：
- 参数：cls（元类本身）、name（类名）、bases（基类元组）、namespace（类属性字典）。
- 可以修改namespace来改变类的属性和方法。
- 必须返回一个类对象。

应用场景：
- 接口检查：自动验证子类是否实现了所有抽象方法。
- 自动注册：将创建的类自动注册到某个注册表中。
- 属性修改：自动添加属性、方法或修改已有属性。
- ORM框架：如Django的Model使用元类将类定义映射到数据库表。
- 单例模式：通过元类控制类的实例化。

注意：元类是深层的魔法，大多数情况下可以用装饰器或混入类替代。"元类就是深度的魔法，99%的用户应该根本不需要关心。" —— Tim Peters

💻 代码示例

# 元类自动注册 class RegistryMeta(type): _registry = {} def __new__(cls, name, bases, namespace): new_cls = super().__new__(cls, name, bases, namespace) if name != 'BasePlugin': cls._registry[name] = new_cls return new_cls @classmethod def get_all(cls): return dict(cls._registry) class BasePlugin(metaclass=RegistryMeta): pass class PluginA(BasePlugin): def run(self): print('PluginA') class PluginB(BasePlugin): def run(self): print('PluginB') print(RegistryMeta.get_all()) # {'PluginA': <class 'PluginA'>, 'PluginB': <class 'PluginB'>}
