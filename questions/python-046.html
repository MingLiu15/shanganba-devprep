📑 题目

Django框架的MTV架构是什么？ORM和中间件如何工作？

💡 参考答案

**Django MTV架构：**
- Model：数据模型，负责数据存取
- Template：模板，负责展示逻辑
- View：视图，负责业务逻辑

**ORM核心：**
1. 模型定义：继承models.Model
2. 查询API：objects.filter(), get(), create()
3. 关系字段：ForeignKey, ManyToManyField, OneToOneField
4. 数据迁移：makemigrations, migrate

**中间件：**
- 请求阶段：process_request
- 视图阶段：process_view
- 响应阶段：process_response
- 异常阶段：process_exception

**信号机制：**
- 内置信号：pre_save, post_save, pre_delete
- 自定义信号：Signal()

💻 代码示例

# models.py from django.db import models class Author(models.Model): name = models.CharField(max_length=100) email = models.EmailField() class Book(models.Model): title = models.CharField(max_length=200) author = models.ForeignKey(Author, on_delete=models.CASCADE) published = models.DateField() # views.py from django.shortcuts import render, get_object_or_404 from .models import Book def book_list(request): books = Book.objects.select_related('author').all() return render(request, 'books/list.html', {'books': books}) def book_detail(request, pk): book = get_object_or_404(Book, pk=pk) return render(request, 'books/detail.html', {'book': book}) # ORM查询示例 # 创建 author = Author.objects.create(name='Alice', email='alice@example.com') # 查询 books = Book.objects.filter(author=author, published__year=2024) # 更新 Book.objects.filter(author=author).update(title='New Title') # 删除 author.delete() # 中间件示例 class CustomMiddleware: def __init__(self, get_response): self.get_response = get_response def __call__(self, request): # 请求前 print(f'请求路径: {request.path}') response = self.get_response(request) # 响应后 response['X-Custom'] = 'value' return response
