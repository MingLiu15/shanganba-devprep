装饰器的原理和应用

Python 中等
标签：基础语法, 装饰器, 高阶函数, 闭包

📑 题目

解释Python装饰器的原理、实现方式及常见应用场景。

💡 参考答案

装饰器本质上是一个高阶函数，接受一个函数作为参数，返回一个新的增强函数。通过@语法糖可以简洁地应用到目标函数上。

实现原理：
- 装饰器是一个接受函数并返回函数的函数（或实现了__call__的类）。
- @decorator等价于func = decorator(func)。
- 使用functools.wraps保留原函数的元信息（__name__、__doc__等）。

常见应用场景：
- 日志记录：记录函数调用信息和执行时间。
- 权限校验：检查用户是否有权限执行操作。
- 缓存：缓存函数返回值（如functools.lru_cache）。
- 重试机制：自动重试失败的函数调用。
- 单例模式：确保类只有一个实例。

高级用法：
- 带参数的装饰器：外层函数接受参数，返回装饰器函数。
- 类装饰器：使用类实现__call__方法。
- 多个装饰器叠加：从下往上依次执行。

💻 代码示例

import functools import time def timer(func): @functools.wraps(func) def wrapper(*args, **kwargs): start = time.time() result = func(*args, **kwargs) print(f'{func.__name__} took {time.time()-start:.4f}s') return result return wrapper # 带参数的装饰器 def retry(max_times=3): def decorator(func): @functools.wraps(func) def wrapper(*args, **kwargs): for i in range(max_times): try: return func(*args, **kwargs) except Exception as e: print(f'第{i+1}次重试: {e}') raise Exception(f'{func.__name__} 执行失败') return wrapper return decorator @retry(max_times=3) def fetch_data(url): pass # 网络请求逻辑
