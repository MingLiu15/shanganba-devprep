单元测试unittest与pytest

Python 中等
标签：unittest, pytest, 单元测试, TDD, 测试框架

📑 题目

比较unittest和pytest的特点。

💡 参考答案

单元测试：验证代码单元（函数/类）的正确性。

unittest：
- 标准库内置
- xUnit风格（Java JUnit类似）
- 类继承TestCase
- 断言方法：assertEqual, assertTrue等

pytest：
- 第三方，更现代
- 函数式，无需类
- 丰富的插件生态
- 更简洁的断言语法
- 自动发现测试
- fixture机制

选择建议：
- 新项目推荐pytest
- 标准库依赖用unittest
- 两者可以混用

💻 代码示例

# unittest示例 import unittest class TestMath(unittest.TestCase): def setUp(self): self.x = 10 def test_add(self): self.assertEqual(1 + 1, 2) self.assertTrue(self.x > 0) def test_divide(self): with self.assertRaises(ZeroDivisionError): 1 / 0 if __name__ == '__main__': unittest.main() # pytest示例 import pytest def add(x, y): return x + y def test_add(): assert add(1, 2) == 3 assert add(-1, 1) == 0 # pytest fixture @pytest.fixture def database(): db = create_db() yield db db.cleanup() def test_query(database): result = database.query('SELECT * FROM users') assert len(result) > 0 # 参数化测试 @pytest.mark.parametrize('x,y,expected', [ (1, 2, 3), (0, 0, 0), (-1, 1, 0), ]) def test_add_param(x, y, expected): assert add(x, y) == expected # 运行测试 # $ pytest test_file.py -v # $ pytest -k 'test_add' # 按名称过滤 # $ pytest --cov=myapp # 覆盖率
