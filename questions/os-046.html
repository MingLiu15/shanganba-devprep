CPU缓存与伪共享（False Sharing）

操作系统 困难
标签：CPU缓存, Cache Line, 伪共享, False Sharing, 内存屏障

📑 题目

什么是CPU缓存？什么是伪共享（False Sharing）？如何避免？

💡 参考答案

**CPU缓存层次：**
- L1 Cache（32-64KB）：每个核心私有，最快
- L2 Cache（256KB-1MB）：每个核心私有
- L3 Cache（数MB-数十MB）：所有核心共享

**Cache Line（缓存行）：**
- CPU缓存以缓存行为单位读取，通常64字节
- 即使只读1字节，也会加载整个缓存行
- 缓存一致性协议（MESI）保证多核数据一致

**伪共享（False Sharing）：**
- 两个无关变量位于同一缓存行
- 不同核心分别修改各自变量
- 导致缓存行频繁失效，性能大幅下降

**MESI协议：**
- M（Modified）：已修改，与内存不一致
- E（Exclusive）：独占，与内存一致
- S（Shared）：共享，多核持有
- I（Invalid）：无效

**避免伪共享的方法：**
1. 缓存行对齐：使用alignas(64)
2. 填充无用字段：手动padding
3. 线程局部存储：ThreadLocal
4. @Contended注解（Java）

💻 代码示例

#include <iostream> #include <vector> #include <thread> #include <atomic> #include <chrono> // 错误示例：伪共享 struct CounterBad { std::atomic<int> count1; // 可能在同一缓存行 std::atomic<int> count2; // 导致伪共享 }; // 正确示例：缓存行对齐，避免伪共享 // 缓存行通常64字节 class alignas(64) PaddedInt { std::atomic<int> value; // 编译器自动填充到64字节 }; struct CounterGood { PaddedInt count1; // 独占一个缓存行 PaddedInt count2; // 独占另一个缓存行 }; // 手动padding方式 class alignas(64) AlignedCounter { std::atomic<int> value; char padding[64 - sizeof(std::atomic<int>)]; }; void benchmark() { const int N = 10000000; // 伪共享测试 CounterBad bad; bad.count1 = 0; bad.count2 = 0; auto start = std::chrono::high_resolution_clock::now(); std::thread t1([&]() { for (int i = 0; i < N; i++) bad.count1++; }); std::thread t2([&]() { for (int i = 0; i < N; i++) bad.count2++; }); t1.join(); t2.join(); auto end = std::chrono::high_resolution_clock::now(); std::cout << "False Sharing: " << std::chrono::duration_cast<std::chrono::milliseconds>(end - start).count() << "ms" << std::endl; // 缓存行对齐测试 CounterGood good; good.count1.value = 0; good.count2.value = 0; start = std::chrono::high_resolution_clock::now(); std::thread t3([&]() { for (int i = 0; i < N; i++) good.count1.value++; }); std::thread t4([&]() { for (int i = 0; i < N; i++) good.count2.value++; }); t3.join(); t4.join(); end = std::chrono::high_resolution_clock::now(); std::cout << "Cache Line Aligned: " << std::chrono::duration_cast<std::chrono::milliseconds>(end - start).count() << "ms" << std::endl; }
