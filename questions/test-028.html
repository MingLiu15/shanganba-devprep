测试左移与测试右移

自动化测试 中等
标签：测试左移, 测试右移, DevOps, 持续测试, 生产环境测试

📑 题目

什么是测试左移和测试右移？如何在实践中应用？

💡 参考答案

**测试左移（Shift Left）：**
- 将测试活动提前到开发早期
- 在需求阶段就考虑可测试性
- 开发编写单元测试
- 代码评审时关注测试
- 收益：缺陷早发现早修复，成本低

**测试右移（Shift Right）：**
- 在生产环境进行测试
- 监控真实用户行为
- A/B测试、金丝雀发布
- 混沌工程
- 收益：验证真实场景，快速反馈

**实践方法：**
- 左移：TDD、静态分析、代码评审
- 右移：监控告警、灰度发布、用户反馈

💻 代码示例

# 测试左移实践 """ 1. 代码提交前检查 - 本地运行单元测试 - 静态代码分析（pylint, mypy） - 代码格式化（black, autopep8） 2. CI/CD流水线 - 提交触发自动化测试 - 测试通过才能合并 - 代码覆盖率门禁 """ # 测试右移实践 """ 1. 生产环境监控 - 错误率监控 - 性能指标监控 - 业务指标监控 2. 灰度发布 - 金丝雀发布 - A/B测试 - 特性开关 3. 混沌工程 - 随机注入故障 - 验证系统韧性 """ # 特性开关示例 class FeatureFlags: def __init__(self): self.flags = { "new_checkout_flow": False, "recommendation_v2": True } def is_enabled(self, flag): return self.flags.get(flag, False) # 使用 ff = FeatureFlags() if ff.is_enabled("new_checkout_flow"): process_new_checkout() else: process_old_checkout()
