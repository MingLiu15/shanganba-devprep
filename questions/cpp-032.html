TCP和UDP的区别

C/C++ 中等
标签：TCP, UDP, 网络, Socket

📑 题目

比较TCP和UDP协议的特点和适用场景。

💡 参考答案

TCP（传输控制协议）：
- 面向连接：三次握手建立连接，四次挥手断开
- 可靠传输：确认应答、重传机制、流量控制、拥塞控制
- 有序传输：序列号保证数据顺序
- 面向字节流：无消息边界
- 开销大：头部20字节，传输效率相对较低
- 适用：HTTP/HTTPS、FTP、邮件等需要可靠传输的场景

UDP（用户数据报协议）：
- 无连接：直接发送数据，无需建立连接
- 不可靠：不保证到达、不保证顺序、无重传
- 面向数据报：保留消息边界
- 开销小：头部仅8字节，传输效率高
- 支持一对一、一对多、多对多通信
- 适用：视频直播、DNS、实时游戏、物联网等对实时性要求高的场景

💻 代码示例

// TCP服务端 int server = socket(AF_INET, SOCK_STREAM, 0); bind(server, ...); listen(server, 5); int client = accept(server, ...); recv(client, buffer, ...); send(client, response, ...); // UDP服务端 int server = socket(AF_INET, SOCK_DGRAM, 0); bind(server, ...); recvfrom(server, buffer, ..., &clientAddr); sendto(server, response, ..., &clientAddr);
