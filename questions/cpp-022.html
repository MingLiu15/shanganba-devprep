内存对齐的原因和规则

C/C++ 中等
标签：内存对齐, alignof, alignas, 性能

📑 题目

什么是内存对齐？为什么要对齐？

💡 参考答案

原因：
1. 硬件限制：某些架构只能访问对齐的地址
2. 性能：未对齐访问可能需要多次内存读取
3. 原子性：某些操作要求对齐

规则：
1. 成员按自身大小对齐（如int按4字节）
2. 结构体整体对齐到最大成员大小
3. 可以通过#pragma pack或alignas修改

控制方法：
- alignof(T)：获取类型对齐要求
- alignas(n)：指定对齐方式
- #pragma pack(n)：按n字节对齐

💻 代码示例

struct S { char c; // 1字节 // 3字节填充 int i; // 4字节，对齐到4 short s; // 2字节 // 2字节填充 }; // 总大小12字节（不是7） cout << alignof(int); // 通常是4 alignas(16) char buffer[64]; // 16字节对齐 #pragma pack(push, 1) struct Packed { char c; int i; // 紧密排列，无填充 }; // 总大小5字节 #pragma pack(pop)
