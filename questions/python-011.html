类变量与实例变量的区别

Python 简单
标签：面向对象, 类变量, 实例变量

📑 题目

解释Python中类变量和实例变量的区别，以及属性查找顺序。

💡 参考答案

类变量：定义在类体中、方法之外，属于类本身，所有实例共享。通过类名或实例名访问。修改类变量会影响所有实例（除非实例已覆盖该属性）。

实例变量：在__init__等方法中通过self.xxx定义，属于具体实例，每个实例独立拥有。只能通过实例名访问。

属性查找顺序（MRO简化版）：
1. 先查找实例的__dict__。
2. 再查找类的__dict__。
3. 再查找基类的__dict__，沿继承链向上查找。

常见陷阱：
- 通过实例修改类变量（如obj.count = 5）不会修改类变量，而是创建同名的实例变量。
- 可变类型的类变量（如list、dict）被实例修改时会真正改变类变量。
- 使用类名来修改类变量才是正确做法：ClassName.var = value。

最佳实践：
- 类变量用于存储所有实例共享的常量或计数器。
- 实例变量用于存储每个实例独有的数据。
- 避免使用可变类型作为类变量的初始值。

💻 代码示例

class Dog: species = 'Canine' # 类变量 count = 0 # 类变量 def __init__(self, name): self.name = name # 实例变量 Dog.count += 1 a = Dog('Buddy') b = Dog('Max') print(a.species) # Canine print(b.count) # 2 # 陷阱：通过实例赋值不会修改类变量 a.species = 'Wolf' print(a.species) # Wolf（实例变量） print(b.species) # Canine（类变量） print(Dog.species) # Canine # 可变类变量的陷阱 class Bad: items = [] # 可变类变量 def add(self, item): self.items.append(item) # 修改了共享的类变量！ x = Bad(); y = Bad() x.add(1) print(y.items) # [1] 被影响了
