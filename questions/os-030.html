容器的实现原理

操作系统 困难
标签：容器, Docker, Namespace, Cgroups, 容器技术

📑 题目

容器（如Docker）是如何实现的？它的核心技术有哪些？

💡 参考答案

容器是一种轻量级的虚拟化技术，与虚拟机不同，容器共享宿主机内核，通过操作系统级隔离实现资源隔离。

核心技术：
1. Namespace（隔离）：
- PID、Mount、Network、UTS、IPC、User、Cgroup Namespace
- 实现进程、网络、文件系统等资源的隔离

2. Cgroups（限制）：
- 限制CPU、内存、IO、网络等资源使用
- 防止单个容器耗尽宿主机资源

3. UnionFS（文件系统）：
- OverlayFS/AUFS实现分层镜像
- 写时复制机制，高效存储

4. 容器运行时：
- runc：底层容器运行时，创建和运行容器
- containerd：管理容器生命周期

容器 vs 虚拟机：
- 容器：共享内核，启动快（秒级），资源占用少
- 虚拟机：独立内核，启动慢（分钟级），资源占用多

容器启动流程：
1. 创建Namespace实现隔离
2. 配置Cgroups限制资源
3. 挂载UnionFS根文件系统
4. 执行容器init进程

💻 代码示例

// 容器启动的核心系统调用流程 // 1. clone创建新进程并指定namespace int pid = clone(child_func, child_stack, CLONE_NEWPID | CLONE_NEWNET | CLONE_NEWNS | CLONE_NEWUTS | CLONE_NEWIPC, NULL); // 2. 配置cgroups // 写入 /sys/fs/cgroup/<controller>/mycontainer/ // 3. pivot_root切换根文件系统 // pivot_root(new_root, put_old); // 4. 执行容器入口程序 execve("/bin/sh", args, env); // 手动模拟容器启动 // $ unshare --pid --net --mount --uts --ipc --fork --mount-proc /bin/bash // $ echo '1000000' > /sys/fs/cgroup/memory/mycontainer/memory.limit_in_bytes // $ echo $$ > /sys/fs/cgroup/memory/mycontainer/cgroup.procs
