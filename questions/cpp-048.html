观察者模式

C/C++ 中等
标签：设计模式, 观察者模式, 发布订阅, 事件系统

📑 题目

解释观察者模式的原理和实现。

💡 参考答案

观察者模式：定义对象间一对多的依赖关系，当一个对象状态改变时，所有依赖它的对象都会收到通知并自动更新。

角色：
1. Subject（被观察者）：维护观察者列表，提供注册/注销/通知接口
2. Observer（观察者）：定义更新接口，接收通知后执行操作

优点：
- 松耦合：被观察者不需要知道观察者的具体类型
- 动态关系：运行时可以添加/移除观察者
- 符合开闭原则

缺点：
- 通知顺序不可控
- 可能导致级联更新（循环通知）
- 调试困难

应用场景：
- GUI事件系统
- MVC架构中Model通知View
- 消息队列/事件总线

💻 代码示例

#include <functional> #include <vector> #include <string> class EventEmitter { std::unordered_map<std::string, std::vector<std::function<void()>>> listeners; public: void on(const std::string& event, std::function<void()> callback) { listeners[event].push_back(callback); } void emit(const std::string& event) { for (auto& cb : listeners[event]) { cb(); } } void off(const std::string& event) { listeners.erase(event); } }; // 使用 EventEmitter emitter; emitter.on("click", []() { /* 处理点击 */ }); emitter.on("click", []() { /* 另一个处理 */ }); emitter.emit("click"); // 通知所有观察者
