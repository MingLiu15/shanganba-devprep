volatile关键字的作用

C/C++ 中等
标签：volatile, 编译器优化, 硬件访问

📑 题目

volatile关键字有什么作用？

💡 参考答案

作用：告诉编译器该变量可能在程序控制之外被修改，禁止对该变量进行优化。

使用场景：
1. 硬件寄存器映射（嵌入式）
2. 多线程共享变量（但volatile不保证原子性，C++11后应使用atomic）
3. 信号处理程序中的变量

注意：
- volatile不保证原子性
- 不保证内存顺序
- C++11后多线程应使用std::atomic

💻 代码示例

// 硬件寄存器 volatile uint32_t* const UART_DR = (uint32_t*)0x4000C000; *UART_DR = 'A'; // 每次都会写入硬件 // 多线程（C++11前） volatile bool stop = false; void thread_func() { while (!stop) { /*...*/ } } // C++11后推荐 #include <atomic> std::atomic<bool> stop2{false};
