进程和线程的区别

中等
标签：进程, 线程, 并发, 操作系统

📑 题目

比较进程（Process）和线程（Thread）的区别。

💡 参考答案

1. 资源拥有：进程是资源分配的基本单位，拥有独立的地址空间、代码段、数据段、堆栈；线程是CPU调度的基本单位，共享进程的资源，只拥有独立的栈和寄存器。
2. 通信方式：进程间通信需要IPC（管道、消息队列、共享内存、信号量、套接字）；线程间可以直接读写共享数据，需要同步机制。
3. 切换开销：进程切换需要切换页表、刷新TLB，开销大；线程切换只需保存/恢复寄存器，开销小。
4. 创建销毁：进程的创建销毁开销远大于线程。
5. 崩溃影响：一个进程崩溃不影响其他进程；一个线程崩溃可能导致整个进程崩溃。
6. 数量限制：进程数量有限（受系统资源限制）；一个进程可以创建更多线程。

💻 代码示例

// 进程创建（Linux） pid_t pid = fork(); if (pid == 0) { // 子进程 } else if (pid > 0) { // 父进程 wait(NULL); // 等待子进程结束 } // 线程创建（C++11） std::thread t([]() { // 线程执行代码 }); t.join(); // 等待线程结束
