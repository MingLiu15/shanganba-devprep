list的底层实现（动态数组）

Python 中等
标签：数据结构, 列表, 动态数组, 底层实现

📑 题目

解释Python列表的底层实现原理，包括动态扩容机制和操作的时间复杂度。

💡 参考答案

Python列表基于动态数组实现，底层是一个连续的内存块，存储指向Python对象的指针（不是对象本身）。

底层结构：
- 一个连续的指针数组，每个元素是一个PyObject*指针。
- 维护已用大小（ob_size）和分配的总容量（allocated）。
- 当已用大小等于总容量时触发扩容。

动态扩容机制：
- 当空间不足时，按增长模式分配新空间：new_allocated = (newsize >> 3) + (newsize < 9 ? 3 : 6) + newsize。
- 大致规律：每次扩容增长约12.5%或25%（取决于当前大小）。
- 扩容时分配新的更大数组，将旧数组元素复制到新数组，释放旧数组。
- 均摊时间复杂度：追加操作均摊O(1)。

时间复杂度：
- 索引访问/修改：O(1)。
- 尾部追加（append）：均摊O(1)。
- 头部/中间插入（insert）：O(n)，需要移动后续元素。
- 尾部删除（pop）：O(1)。
- 头部/中间删除（pop(index)）：O(n)。
- 查找（in操作）：O(n)。

与链表的区别：
- 列表是动态数组，随机访问O(1)，插入删除O(n)。
- 链表随机访问O(n)，已知位置插入删除O(1)。
- 列表缓存友好（连续内存），实际性能通常优于链表。

💻 代码示例

# 列表操作时间复杂度 import timeit # O(1) 索引访问 print(timeit.timeit('lst[500]', setup='lst=list(range(1000))', number=1000000)) # O(1) 尾部追加（均摊） print(timeit.timeit('lst.append(1)', setup='lst=[]', number=1000000)) # O(n) 头部插入 print(timeit.timeit('lst.insert(0, 1)', setup='lst=list(range(1000))', number=10000)) # O(n) 中间删除 print(timeit.timeit('lst.pop(500)', setup='lst=list(range(1000))', number=10000)) # 预分配空间提升性能 def append_no_prealloc(n): lst = [] for i in range(n): lst.append(i) def append_with_prealloc(n): lst = [None] * n # 预分配 for i in range(n): lst[i] = i # 预分配版本更快（减少扩容次数）
