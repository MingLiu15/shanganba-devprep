同步与互斥

操作系统 中等
标签：同步, 互斥, 信号量, 互斥锁, 临界区

📑 题目

同步和互斥有什么区别？如何实现？

💡 参考答案

互斥（Mutual Exclusion）：
- 多个进程不能同时进入临界区
- 保证同一时刻只有一个进程访问共享资源
- 关系：竞争关系

同步（Synchronization）：
- 进程按一定顺序执行
- 保证事件的先后顺序
- 关系：协作关系

实现机制：
1. 互斥锁（Mutex）：保护临界区，同一时刻只有一个持有者
2. 信号量（Semaphore）：
- 二值信号量：实现互斥
- 计数信号量：实现同步（如生产者-消费者）
3. 条件变量（Condition Variable）：等待某个条件成立

经典问题：
- 生产者-消费者：用信号量同步
- 读者-写者：读写锁
- 哲学家就餐：避免死锁

💻 代码示例

// 信号量实现生产者-消费者 sem_t empty, full, mutex; sem_init(&empty, 0, N); // 空槽位 sem_init(&full, 0, 0); // 已有产品 sem_init(&mutex, 0, 1); // 互斥 void producer() { sem_wait(&empty); // 等待空槽 sem_wait(&mutex); // 加锁 buffer[in++] = item; sem_post(&mutex); // 解锁 sem_post(&full); // 通知消费者 } void consumer() { sem_wait(&full); // 等待产品 sem_wait(&mutex); item = buffer[out++]; sem_post(&mutex); sem_post(&empty); // 通知生产者 }
