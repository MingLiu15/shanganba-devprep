信号机制

操作系统 中等
标签：信号, signal, 异步, SIGINT, SIGKILL

📑 题目

Linux信号机制的工作原理是什么？

💡 参考答案

信号是软件中断，用于通知进程异步事件。

常见信号：
- SIGINT(2)：Ctrl+C中断
- SIGKILL(9)：强制终止（不能被捕获或忽略）
- SIGTERM(15)：请求终止（可被捕获）
- SIGSEGV(11)：段错误
- SIGCHLD(17)：子进程状态变化
- SIGSTOP(19)：暂停进程
- SIGCONT(18)：继续暂停的进程

信号处理方式：
1. 默认处理：终止/暂停/忽略
2. 忽略信号：signal(SIGINT, SIG_IGN)
3. 捕获信号：注册信号处理函数

信号特点：
- 不能传递数据（只传递信号编号）
- 不能保证实时性
- 同种信号不排队（多次发送只接收一次）
- 处理函数中只能调用异步安全函数

不可捕获/忽略的信号：SIGKILL、SIGSTOP

💻 代码示例

#include <signal.h> #include <unistd.h> // 信号处理函数 void sigint_handler(int signum) { printf("收到SIGINT信号\n"); } int main() { // 注册信号处理 signal(SIGINT, sigint_handler); // sigaction（更推荐，功能更强） struct sigaction sa; sa.sa_handler = sigint_handler; sigemptyset(&sa.sa_mask); sa.sa_flags = 0; sigaction(SIGINT, &sa, NULL); // 发送信号 kill(getpid(), SIGUSR1); raise(SIGUSR1); while(1) pause(); // 等待信号 }
