C++11/14 异常处理与noexcept

C/C++ 中等
标签：C++11, 异常处理, noexcept, 异常安全, 现代C++

📑 题目

C++11的异常处理机制有什么改进？noexcept关键字的作用是什么？

💡 参考答案

**异常处理基础：**
- try/catch/throw机制
- 异常类层次结构（std::exception）
- 栈展开（stack unwinding）

**C++11改进：**
- noexcept关键字替代throw()
- 异常规范被弃用（dynamic exception specification）
- std::nested_exception和std::throw_with_nested
- std::current_exception和std::rethrow_exception

**noexcept关键字：**
- 承诺函数不抛出异常
- 用于移动构造函数、移动赋值运算符
- 编译器可以优化noexcept函数
- 条件noexcept：noexcept(expression)

**异常安全保证：**
- 基本保证：异常发生时对象处于有效状态
- 强保证：异常发生时状态不变
- 不抛异常保证：承诺不抛出异常

💻 代码示例

#include <iostream> #include <stdexcept> #include <vector> // noexcept函数 void safe_function() noexcept { // 承诺不抛出异常 } // 条件noexcept template<typename T> void swap(T& a, T& b) noexcept(noexcept(T(std::declval<T>())) && noexcept(std::declval<T>() = std::declval<T>())) { T temp = std::move(a); a = std::move(b); b = std::move(temp); } // 异常类层次 class MyException : public std::exception { const char* msg; public: MyException(const char* m) : msg(m) {} const char* what() const noexcept override { return msg; } }; // 异常安全示例 class Resource { int* data; public: Resource() : data(new int[100]) { std::cout << "Resource acquired" << std::endl; } ~Resource() { delete[] data; std::cout << "Resource released" << std::endl; } // 移动操作标记为noexcept Resource(Resource&& other) noexcept : data(other.data) { other.data = nullptr; } Resource& operator=(Resource&& other) noexcept { if (this != &other) { delete[] data; data = other.data; other.data = nullptr; } return *this; } }; void risky_function(bool should_throw) { Resource r; // RAII保证资源释放 if (should_throw) { throw MyException("Something went wrong"); } } int main() { // 基本异常处理 try { risky_function(true); } catch (const MyException& e) { std::cout << "Caught: " << e.what() << std::endl; } catch (const std::exception& e) { std::cout << "Standard exception: " << e.what() << std::endl; } catch (...) { std::cout << "Unknown exception" << std::endl; } // noexcept检查 std::cout << "safe_function is noexcept: " << noexcept(safe_function()) << std::endl; return 0; }
