测试替身（Test Doubles）

自动化测试 中等
标签：Mock, Stub, Fake, Spy, 测试替身

📑 题目

什么是测试替身？Mock、Stub、Fake、Spy有什么区别？

💡 参考答案

测试替身：测试中替代真实依赖的对象。

1. Dummy（哑元）：仅填充参数，不会被使用
2. Stub（存根）：提供预设的返回值，控制执行路径
3. Fake（伪造）：有真实实现，但简化/优化
4. Spy（间谍）：记录调用信息供验证
5. Mock（模拟）：预设期望和返回值，验证交互行为

选择建议：
- 优先使用Fake（简单且真实）
- 需要验证交互时用Mock/Spy
- 只需控制返回值用Stub

💻 代码示例

from unittest.mock import Mock, patch import pytest # 1. Stub - 提供固定返回值 class PaymentGatewayStub: def charge(self, amount): return {'success': True, 'transaction_id': 'stub-123'} def test_order_processing_with_stub(): order = Order(PaymentGatewayStub()) result = order.process_payment(100) assert result['success'] # 2. Fake - 简化实现 class InMemoryDatabase: def __init__(self): self.data = {} def save(self, key, value): self.data[key] = value def get(self, key): return self.data.get(key) def test_user_repository_with_fake(): repo = UserRepository(InMemoryDatabase()) repo.save(User(id=1, name='Alice')) user = repo.get(1) assert user.name == 'Alice' # 3. Mock - 验证交互 def test_user_service_sends_welcome_email(): email_service = Mock() user_service = UserService(email_service) user_service.register('alice@example.com', 'password') email_service.send_welcome.assert_called_once_with('alice@example.com') # 4. 使用patch进行依赖注入 @patch('mymodule.requests.get') def test_fetch_data(mock_get): mock_get.return_value.json.return_value = {'data': 'test'} result = fetch_data('http://api.example.com') assert result == {'data': 'test'} mock_get.assert_called_once_with('http://api.example.com')
