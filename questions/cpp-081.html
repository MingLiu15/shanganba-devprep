C++11 override与final

C/C++ 简单
标签：C++11, override, final, 虚函数, 现代C++

📑 题目

C++11的override和final关键字有什么作用？

💡 参考答案

**override关键字：**
- 显式标记重写基类虚函数
- 编译器检查签名是否匹配
- 防止因签名不匹配导致的隐藏bug

**final关键字：**
- 修饰类：禁止继承
- 修饰虚函数：禁止重写
- 用于设计不可扩展的类或方法

**使用建议：**
- 所有重写虚函数都加上override
- 需要阻止继承的类使用final
- 提高代码可读性和安全性

💻 代码示例

#include <iostream> class Base { public: virtual void foo(int x); virtual void bar(); virtual void baz() final; // 禁止重写 }; class Derived : public Base { public: // 正确：显式重写 void foo(int x) override; // 错误：参数不匹配，override会报错 // void foo(double x) override; // 编译错误 // 错误：baz是final，不能重写 // void baz() override; // 编译错误 void bar() override; }; // final类：禁止继承 class Sealed final { // ... }; // 错误：不能继承final类 // class TryInherit : public Sealed {}; // 编译错误 // 实际应用示例 class Animal { public: virtual void makeSound() const = 0; virtual ~Animal() = default; }; class Dog final : public Animal { // Dog不能被继承 public: void makeSound() const override { std::cout << "Woof!" << std::endl; } }; class Cat : public Animal { public: void makeSound() const override { std::cout << "Meow!" << std::endl; } virtual void purr() final; // 子类不能重写purr }; int main() { Dog d; d.makeSound(); Cat c; c.makeSound(); return 0; }
