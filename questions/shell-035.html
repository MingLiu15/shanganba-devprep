进程替换与Here Document高级用法

Shell脚本 困难
标签：进程替换, Here Document, Here String, 高级重定向, Process Substitution

📑 题目

什么是进程替换？Here Document有哪些高级用法？

💡 参考答案

**进程替换（Process Substitution）：**
- `<(command)`：将命令输出作为文件读取
- `>(command)`：将写入的内容传递给命令处理
- 本质上是创建临时管道文件
- 优势：可以像操作文件一样操作命令输出

**Here Document：**
- `<<EOF`：标准here document
- `<<-EOF`：去除前导制表符（用于缩进）
- `<<'EOF'`：禁用变量展开和命令替换
- `<<< string`：Here String（单行）

**应用场景：**
- 进程替换：比较两个命令输出、多输入源
- Here Document：多行输入、SQL脚本、配置文件生成

💻 代码示例

# 进程替换示例 print(''' # 比较两个命令输出（无需临时文件） diff <(ls dir1) <(ls dir2) # 同时读取多个命令输出 cat <(echo "Header") <(cat data.txt) <(echo "Footer") # 结合while循环 while read line; do echo "Processing: $line" done < <(grep "pattern" file.txt) # Here Document标准用法 cat << EOF This is line 1 This is line 2 EOF # 去除前导制表符 cat <<- EOF \tThis line has no leading tab EOF # 禁用变量展开 cat << 'EOF' This $variable will not expand EOF # Here String grep "pattern" <<< "this is a test string" ''')
