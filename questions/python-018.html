上下文管理器（with语句）

Python 中等
标签：进阶特性, 上下文管理器, with语句, 资源管理

📑 题目

解释Python上下文管理器的原理、实现方式及应用场景。

💡 参考答案

上下文管理器是一种用于管理资源的协议，通过with语句使用，确保资源在使用后被正确释放，即使发生异常。

实现方式：
1. 基于类的实现：实现__enter__和__exit__方法。
- __enter__：进入上下文时执行，返回值赋给as后面的变量。
- __exit__：退出上下文时执行，负责清理资源。参数接收异常信息。
- __exit__返回True可以抑制异常。

2. 基于生成器的实现：使用contextlib.contextmanager装饰器。
- yield之前的代码相当于__enter__。
- yield的值赋给as后面的变量。
- yield之后的代码相当于__exit__，即使发生异常也会执行。

应用场景：
- 文件操作：自动关闭文件。
- 数据库连接：自动关闭连接和游标。
- 线程锁：自动获取和释放锁。
- 临时改变状态：如临时改变工作目录。
- 事务管理：自动提交或回滚。

contextlib模块还提供了closing()、suppress()、redirect_stdout()等实用工具。

💻 代码示例

# 类方式实现 class Timer: def __enter__(self): import time self.start = time.time() return self def __exit__(self, exc_type, exc_val, exc_tb): import time print(f'耗时: {time.time()-self.start:.4f}s') return False # 不抑制异常 with Timer(): sum(i*i for i in range(1000000)) # contextmanager方式 from contextlib import contextmanager @contextmanager def file_manager(path, mode='r'): f = open(path, mode) try: yield f finally: f.close() with file_manager('test.txt', 'w') as f: f.write('hello') # suppress 忽略异常 from contextlib import suppress with suppress(FileNotFoundError): os.remove('nonexistent.txt')
