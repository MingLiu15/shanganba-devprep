Linux文件权限模型

操作系统 简单
标签：文件权限, rwx, ugo, suid, sticky, Linux

📑 题目

Linux文件权限模型是如何工作的？

💡 参考答案

权限分为三组（ugo）：
- u（user）：文件所有者
- g（group）：文件所属组
- o（others）：其他用户

每组三个权限（rwx）：
- r（read）：读取（4）
- w（write）：写入（2）
- x（execute）：执行（1）

常见权限：
- 755：rwxr-xr-x（所有者全权限，其他读和执行）
- 644：rw-r--r--（所有者读写，其他只读）
- 700：rwx------（仅所有者全权限）

特殊权限：
- SUID（4xxx）：执行时以文件所有者身份运行（如/usr/bin/passwd）
- SGID（2xxx）：执行时以文件所属组身份运行；目录下新建文件继承组
- Sticky（1xxx）：目录下文件只有所有者和root能删除（如/tmp）

查看权限：
- ls -la：显示权限、所有者、组
- stat：详细信息

修改权限：
- chmod 755 file
- chown user:group file
- umask：设置默认权限掩码

💻 代码示例

// 查看权限 // $ ls -la // -rwxr-xr-x 1 user group 4096 Jan 1 file // |||||||||| // |||||||||其他: r-x // |||||||组: r-x // |||||所有者: rwx // ||||类型: -（普通文件） // 特殊权限 // $ ls -la /usr/bin/passwd // -rwsr-xr-x 1 root root ... passwd // ^ // SUID位（s替代x） // $ ls -la /tmp // drwxrwxrwt 10 root root ... tmp // ^ // Sticky位（t替代x） // 修改权限 // chmod 755 file.sh // chmod u+x file.sh // 给所有者加执行权限 // chmod g-w file.sh // 去掉组写权限 // chown user:group file // 修改所有者和组 // umask // umask 0022 // 新文件默认权限 0644 (0666-0022) // umask 0002 // 新文件默认权限 0664
