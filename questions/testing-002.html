单元测试的核心原则

自动化测试 简单
标签：单元测试, FIRST原则, AAA模式, 测试驱动开发

📑 题目

编写好的单元测试应遵循哪些原则？

💡 参考答案

FIRST原则：
1. Fast（快速）：测试应快速执行，频繁运行
2. Isolated（独立）：测试间不依赖，可单独运行
3. Repeatable（可重复）：任何环境结果一致
4. Self-validating（自验证）：明确通过/失败，无需人工检查
5. Timely（及时）：与代码同时编写（TDD）

AAA模式（Arrange-Act-Assert）：
- Arrange：准备测试数据和依赖
- Act：执行被测操作
- Assert：验证结果

其他最佳实践：
1. 一个测试只验证一个概念
2. 测试名称清晰描述场景和预期
3. 避免逻辑（if/for）
4. 使用有意义的断言消息
5. 保持测试代码简洁

💻 代码示例

import pytest # AAA模式示例 def test_withdraw_sufficient_balance(): # Arrange account = Account(balance=1000) # Act result = account.withdraw(500) # Assert assert result is True assert account.balance == 500 # 好的测试命名 def test_withdraw_insufficient_balance_returns_false(): account = Account(balance=100) result = account.withdraw(200) assert result is False # 避免多个断言测试不同概念（应拆分） def test_account_deposit_100_increases_balance_by_100(): account = Account(balance=0) account.deposit(100) assert account.balance == 100 def test_account_deposit_50_increases_balance_by_50(): account = Account(balance=0) account.deposit(50) assert account.balance == 50
