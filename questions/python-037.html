WSGI与ASGI

Python 中等
标签：WSGI, ASGI, Web服务器, 同步, 异步

📑 题目

解释WSGI和ASGI的区别及应用场景。

💡 参考答案

WSGI（Web Server Gateway Interface）：
- Python Web服务器网关接口标准
- 同步接口，一次处理一个请求
- 被Django、Flask等框架广泛使用

ASGI（Asynchronous Server Gateway Interface）：
- WSGI的异步继任者
- 支持异步、非阻塞IO
- 支持HTTP/1.1、HTTP/2、WebSocket
- 被FastAPI、Django Channels等使用

对比：

| 特性 | WSGI | ASGI |
|------|------|------|
| 同步/异步 | 同步 | 异步 |
| 协议支持 | HTTP/1.1 | HTTP/1.1, HTTP/2, WebSocket |
| 并发处理 | 多进程/线程 | 单线程异步事件循环 |
| 适用场景 | 传统Web应用 | 实时应用、高并发 |

选择建议：
- 传统CRUD应用→WSGI（Django/Flask）
- 实时通信、高并发→ASGI（FastAPI/Django Channels）

💻 代码示例

# WSGI应用示例（同步） def wsgi_application(environ, start_response): # environ: 包含请求信息的字典 # start_response: 设置响应状态的回调 status = '200 OK' headers = [('Content-Type', 'text/html')] start_response(status, headers) return [b'Hello, WSGI!'] # ASGI应用示例（异步） async def asgi_application(scope, receive, send): # scope: 连接信息 # receive: 接收消息的异步函数 # send: 发送消息的异步函数 await send({ 'type': 'http.response.start', 'status': 200, 'headers': [[b'content-type', b'text/html']], }) await send({ 'type': 'http.response.body', 'body': b'Hello, ASGI!', }) # 使用gunicorn运行WSGI # $ gunicorn myapp:wsgi_application # 使用uvicorn运行ASGI # $ uvicorn myapp:asgi_application # FastAPI（基于ASGI） from fastapi import FastAPI app = FastAPI() @app.get("/") async def root(): return {"message": "Hello ASGI"} # Django支持ASGI（asgi.py） import os from django.core.asgi import get_asgi_application os.environ.setdefault('DJANGO_SETTINGS_MODULE', 'myproject.settings') application = get_asgi_application()
