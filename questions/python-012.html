__new__与__init__的区别

Python 中等
标签：面向对象, 魔术方法, 对象创建

📑 题目

解释Python中__new__和__init__的区别、执行顺序及各自的使用场景。

💡 参考答案

__new__：类方法（第一个参数是cls），负责创建并返回实例对象。是真正的构造方法，控制对象的创建过程。必须返回一个实例。在__init__之前调用。

__init__：实例方法（第一个参数是self），负责初始化已创建的实例。不返回值（返回None）。在对象创建后调用，设置实例属性。

执行顺序：__new__ -> __init__。如果__new__不返回当前类的实例，__init__不会被调用。

__new__的使用场景：
- 实现单例模式：控制只创建一个实例。
- 继承不可变类型（如str、int、tuple）：需要在创建时修改值。
- 自定义元类中控制类的创建过程。
- 工厂模式：根据参数返回不同类的实例。

区别总结：
- __new__负责创建对象，__init__负责初始化对象。
- __new__是类方法，__init__是实例方法。
- __new__必须返回实例，__init__不返回值。
- __new__只在需要控制对象创建时才需要重写。
