宏定义与const/inline的区别

C/C++ 中等
标签：宏, const, inline, 预处理器

📑 题目

比较#define宏与const常量、inline函数的优劣。

💡 参考答案

宏定义的问题：
1. 无类型检查，不安全
2. 无作用域，可能冲突
3. 调试困难，预处理后就没了
4. 可能产生副作用

const优势：
1. 有类型，编译器检查
2. 有作用域
3. 可调试
4. 可用于数组大小（C++）

inline优势：
1. 类型安全
2. 可调试
3. 有作用域
4. 避免宏的副作用

💻 代码示例

// 宏的问题 #define MAX(a,b) ((a)>(b)?(a):(b)) int x = 5, y = 3; MAX(x++, y); // x被递增两次！ // 更好的替代 template<typename T> inline T max(T a, T b) { return a > b ? a : b; } const int SIZE = 100; // 替代 #define SIZE 100
