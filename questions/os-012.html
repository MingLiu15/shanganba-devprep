管道与消息队列

操作系统 中等
标签：管道, 消息队列, IPC, 同步

📑 题目

管道和消息队列有什么区别？

💡 参考答案

管道（Pipe）：
- 面向字节流，无消息边界
- 半双工（单向），大小受限（通常64KB）
- 读取后数据就消失了
- 适合父子进程间的简单数据流

命名管道（FIFO）：
- 在文件系统中存在路径名
- 无亲缘关系的进程也能通信
- 数据读后消失

消息队列（Message Queue）：
- 面向消息，有类型和边界
- 可以按类型选择性读取
- 内核维护，进程退出后消息仍在
- 支持优先级
- 大小受系统限制（msgmax）

区别：
1. 数据格式：管道是字节流，消息队列有结构
2. 生命周期：管道随进程消失，消息队列持久化
3. 读取方式：管道先进先出，消息队列可按类型
4. 适用场景：管道适合简单数据流，消息队列适合结构化通信

💻 代码示例

// 管道 int fd[2]; pipe(fd); pid_t pid = fork(); if (pid == 0) { close(fd[1]); read(fd[0], buf, 128); } else { close(fd[0]); write(fd[1], "hello", 5); } // 消息队列 #include <sys/msg.h> struct msgbuf { long mtype; // 消息类型 char mtext[100]; // 消息内容 }; int msqid = msgget(IPC_PRIVATE, 0666 | IPC_CREAT); struct msgbuf msg = {1, "hello"}; msgsnd(msqid, &msg, sizeof(msg.mtext), 0); msgrcv(msqid, &msg, sizeof(msg.mtext), 1, 0);
