Linux Namespace（命名空间）

操作系统 困难
标签：Namespace, 容器, 隔离, Linux内核, Docker

📑 题目

什么是Linux Namespace？它如何实现资源隔离？

💡 参考答案

Namespace是Linux内核提供的一种资源隔离机制，是容器技术（如Docker）的基础。

七大Namespace类型：
1. PID Namespace：隔离进程ID
- 容器内PID从1开始，与宿主机隔离
- 容器内只能看到自己的进程

2. Mount Namespace：隔离文件系统挂载点
- 容器有自己的根文件系统
- 挂载操作不影响宿主机

3. Network Namespace：隔离网络资源
- 独立的网络接口、IP地址、路由表
- 容器间网络隔离

4. UTS Namespace：隔离主机名和域名
- 容器可以有自己的hostname

5. IPC Namespace：隔离进程间通信
- 隔离共享内存、消息队列、信号量

6. User Namespace：隔离用户和组ID
- 容器内root映射到宿主机普通用户
- 提升安全性

7. Cgroup Namespace：隔离cgroup视图
- 容器内只能看到自己的cgroup

使用方式：
- clone()系统调用：创建新进程时指定namespace
- unshare()：当前进程进入新的namespace
- setns()：将进程加入已存在的namespace

💻 代码示例

// 使用unshare创建新的namespace #define _GNU_SOURCE #include <sched.h> #include <unistd.h> int main() { // 创建新的PID、Network、Mount namespace unshare(CLONE_NEWPID | CLONE_NEWNET | CLONE_NEWNS); // 在新namespace中执行shell char* args[] = {"/bin/bash", NULL}; execvp(args[0], args); } // 命令行操作 // $ sudo unshare --pid --net --mount --fork /bin/bash // $ echo $$ // 显示PID为1 // 查看进程的namespace // $ ls -la /proc/<pid>/ns/
