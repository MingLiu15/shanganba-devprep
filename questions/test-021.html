📑 题目

如何进行数据库测试？有哪些数据验证方法？

💡 参考答案

**数据库测试类型：**
1. **Schema测试**：验证表结构、字段类型、约束
2. **数据完整性测试**：主键、外键、唯一性约束
3. **CRUD测试**：增删改查操作验证
4. **事务测试**：ACID特性验证
5. **性能测试**：查询优化、索引效果
6. **迁移测试**：数据库升级、数据迁移

**数据验证方法：**
- 行数验证：验证操作影响的行数
- 数据内容验证：验证字段值正确性
- 状态验证：验证数据状态变化
- 关联验证：验证外键关联数据

**测试策略：**
1. 使用测试数据库，避免污染生产数据
2. 每个测试后回滚事务
3. 使用Fixture准备测试数据
4. 验证数据库触发器和存储过程

💻 代码示例

import pytest import sqlite3 from contextlib import contextmanager @contextmanager def get_db_connection(): conn = sqlite3.connect(":memory:") conn.row_factory = sqlite3.Row yield conn conn.close() @pytest.fixture def db(): with get_db_connection() as conn: conn.execute("CREATE TABLE users (id INTEGER PRIMARY KEY AUTOINCREMENT, username TEXT UNIQUE NOT NULL, email TEXT NOT NULL)") conn.commit() yield conn class TestDatabaseOperations: def test_insert_user(self, db): cursor = db.execute("INSERT INTO users (username, email) VALUES (?, ?)", ("testuser", "test@example.com")) db.commit() assert cursor.lastrowid == 1 result = db.execute("SELECT * FROM users WHERE id = ?", (cursor.lastrowid,)).fetchone() assert result["username"] == "testuser" def test_unique_constraint(self, db): db.execute("INSERT INTO users (username, email) VALUES (?, ?)", ("user1", "user1@example.com")) db.commit() with pytest.raises(sqlite3.IntegrityError): db.execute("INSERT INTO users (username, email) VALUES (?, ?)", ("user1", "user2@example.com")) db.commit() def test_transaction_rollback(self, db): try: db.execute("BEGIN") db.execute("INSERT INTO users (username, email) VALUES (?, ?)", ("user2", "user2@example.com")) raise Exception("模拟错误") except: db.rollback() result = db.execute("SELECT * FROM users WHERE username = ?", ("user2",)).fetchone() assert result is None
