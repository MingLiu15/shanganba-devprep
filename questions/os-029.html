UnionFS（联合文件系统）

操作系统 困难
标签：UnionFS, OverlayFS, 容器, 文件系统, Docker

📑 题目

什么是UnionFS？它在容器技术中有什么作用？

💡 参考答案

UnionFS（联合文件系统）是一种分层、轻量级的文件系统，可以将多个目录（层）合并挂载到一个统一的视图。

核心概念：
1. 分层结构：
- 底层（Lower Layer）：只读层，通常是基础镜像
- 顶层（Upper Layer）：可写层，容器运行时修改
- 工作目录（Workdir）：OverlayFS内部使用
- 合并视图（Merged）：统一呈现所有层的内容

2. 写时复制（Copy-on-Write）：
- 读取：从上到下查找文件
- 修改：将文件从只读层复制到可写层再修改
- 删除：在可写层创建whiteout标记

常见实现：
- AUFS：早期Docker使用
- OverlayFS：现代Linux默认，性能更好
- OverlayFS2：OverlayFS的改进版

容器中的应用：
- 镜像层共享：多个容器共享基础镜像层
- 容器隔离：每个容器有独立的可写层
- 快速启动：无需复制整个镜像

💻 代码示例

// OverlayFS挂载示例 // $ sudo mount -t overlay overlay \ // -o lowerdir=/lower,upperdir=/upper,workdir=/work \ // /merged // 目录结构说明 // /lower: 只读层（基础镜像） // /upper: 可写层（容器修改） // /work: 工作目录（OverlayFS内部使用） // /merged: 合并后的视图 // 查看Docker容器文件系统 // $ docker inspect <container_id> | grep -A 5 GraphDriver // C语言使用mount系统调用 #include <sys/mount.h> mount("overlay", "/merged", "overlay", 0, "lowerdir=/lower,upperdir=/upper,workdir=/work"); // 查看镜像层 // $ ls /var/lib/docker/overlay2/
