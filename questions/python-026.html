多线程共享数据与线程安全

Python 中等
标签：并发, 线程安全, 锁, 同步

📑 题目

解释Python多线程中共享数据的问题及线程安全的解决方案。

💡 参考答案

多线程共享数据面临的问题：
- 竞态条件（Race Condition）：多个线程同时读写共享数据，导致结果不确定。
- 由于GIL的存在，Python的线程切换可能发生在任何字节码指令之间。
- 即使简单的count += 1操作也不是原子性的（包含读取、计算、写回三步）。

同步原语：
1. Lock（互斥锁）：最基本的锁，一次只允许一个线程访问。acquire()获取锁，release()释放锁。推荐使用with语句。

2. RLock（可重入锁）：同一线程可以多次获取锁而不会死锁。适用于递归调用。

3. Semaphore（信号量）：控制同时访问资源的线程数量。

4. Condition（条件变量）：线程间通信，wait()等待条件，notify()/notify_all()通知。

5. Event：简单的线程间信号通知，set()设置信号，wait()等待信号。

6. Barrier：让一组线程等待所有线程到达某个点后再一起继续。

线程安全的数据结构：
- queue.Queue：线程安全的FIFO队列。
- queue.LifoQueue：线程安全的LIFO队列。
- queue.PriorityQueue：线程安全的优先队列。
- collections.deque：append和popleft是原子操作，但其他操作不是。

最佳实践：尽量减少共享数据的范围，使用队列进行线程间通信而非共享状态。

💻 代码示例

import threading import queue # Lock 使用 lock = threading.Lock() counter = 0 def increment(): global counter for _ in range(100000): with lock: # 自动获取和释放 counter += 1 threads = [threading.Thread(target=increment) for _ in range(5)] for t in threads: t.start() for t in threads: t.join() print(counter) # 500000（线程安全） # 线程安全队列 q = queue.Queue() def producer(): for i in range(10): q.put(i) print(f'生产: {i}') def consumer(): while True: item = q.get() print(f'消费: {item}') q.task_done() threading.Thread(target=producer).start() threading.Thread(target=consumer, daemon=True).start() q.join() # 等待所有任务完成
