测试覆盖率与质量门禁

自动化测试 中等
标签：覆盖率, Code Coverage, 质量门禁, SonarQube, CI质量

📑 题目

如何设置测试覆盖率和质量门禁？

💡 参考答案

测试覆盖率指标：
1. 行覆盖率（Line Coverage）：执行的代码行比例
2. 分支覆盖率（Branch Coverage）：执行的分支比例
3. 函数覆盖率（Function Coverage）：执行的函数比例
4. 语句覆盖率（Statement Coverage）：执行的语句比例

质量门禁策略：
1. 覆盖率阈值：最低覆盖率要求（如80%）
2. 增量覆盖：新代码必须100%覆盖
3. 静态分析：代码规范、复杂度、漏洞
4. 测试通过率：所有测试必须通过

常用工具：
- coverage.py：Python覆盖率
- JaCoCo：Java覆盖率
- SonarQube：代码质量平台

💻 代码示例

# pytest-cov配置 # pytest --cov=src --cov-report=html --cov-report=xml # .coveragerc配置 # [run] # source = src # omit = */tests/*, */venv/* # [report] # fail_under = 80 # 覆盖率检查脚本 import coverage import sys cov = coverage.Coverage() cov.load() if cov.get_total_coverage() < 80.0: print('Coverage below 80%') sys.exit(1) # SonarQube配置 # sonar.projectKey=myproject # sonar.sources=src # sonar.python.coverage.reportPaths=coverage.xml # 增量覆盖检查 # diff-cover coverage.xml --compare-branch=main --fail-under=100 # 代码复杂度检查 # radon cc src -a -nc
