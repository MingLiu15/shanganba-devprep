内存分配方式

操作系统 中等
标签：内存分配, 伙伴系统, slab, malloc, 堆

📑 题目

操作系统如何管理内存分配？

💡 参考答案

连续内存分配：
1. 首次适应（First Fit）：从头找第一个够大的空闲块
2. 最佳适应（Best Fit）：找最小的够大的空闲块
3. 最差适应（Worst Fit）：找最大的空闲块

碎片问题：
- 外部碎片：空闲块太小无法分配（可通过紧凑解决）
- 内部碎片：分配的块比需要的大（分区大小固定时）

Linux内存管理：
1. 伙伴系统（Buddy System）：
- 以2的幂次方分配物理页
- 合并相邻空闲页，减少外部碎片

2. Slab分配器：
- 针对内核小对象（如task_struct、inode）
- 避免频繁分配/释放的开销
- 对象缓存复用

3. 用户空间（glibc malloc）：
- brk：分配小内存，调整堆顶
- mmap：分配大内存（>128KB），映射新区域
- ptmalloc：维护空闲链表（fastbin, smallbin, largebin）

💻 代码示例

// 伙伴系统（简化） // 物理页以2^n分配 // 请求4页 → 找8页块 → 分裂为两个4页 // 释放4页 → 检查伙伴是否也空闲 → 合并为8页 // malloc内部实现 void* my_malloc(size_t size) { if (size > 128 * 1024) return mmap(NULL, size, PROT_READ|PROT_WRITE, MAP_PRIVATE|MAP_ANONYMOUS, -1, 0); else return sbrk(size); // 或brk调整堆顶 } // 内存对齐 // malloc返回的地址保证8/16字节对齐
