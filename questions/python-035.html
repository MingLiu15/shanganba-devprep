Python中的鸭子类型

Python 中等
标签：鸭子类型, 动态类型, 多态, Pythonic

📑 题目

什么是鸭子类型？Python中如何体现？

💡 参考答案

鸭子类型（Duck Typing）：
- 一种动态类型风格
- "如果它走起来像鸭子，叫起来像鸭子，那它就是鸭子"
- 不关注对象类型，只关注对象是否有需要的方法/属性

与静态语言对比：
- Java/C++：编译期检查类型，必须继承同一接口
- Python：运行期检查，只要有相同方法即可

Python体现：
1. 无需显式实现接口
2. 依赖注入简单
3. 迭代协议（__iter__/__next__）
4. 上下文管理协议（__enter__/__exit__）

优点：
- 代码更灵活、简洁
- 易于扩展

缺点：
- 运行时才发现错误
- 需要良好的文档和测试

💻 代码示例

# 鸭子类型示例 class Duck: def quack(self): return "Quack!" def fly(self): return "Flapping wings" class Person: def quack(self): return "I'm quacking like a duck!" def fly(self): return "I'm flapping my arms!" def make_it_quack_and_fly(thing): # 不检查类型，只检查是否有方法 print(thing.quack()) print(thing.fly()) # 两者都可以传入 duck = Duck() person = Person() make_it_quack_and_fly(duck) # 正常工作 make_it_quack_and_fly(person) # 同样正常工作！ # 迭代协议（鸭子类型经典例子） class MyRange: def __init__(self, n): self.n = n self.i = 0 def __iter__(self): return self def __next__(self): if self.i >= self.n: raise StopIteration val = self.i self.i += 1 return val # 不需要继承任何接口，只要有__iter__和__next__就能迭代 for i in MyRange(5): print(i) # 0 1 2 3 4
