多继承与MRO（方法解析顺序）

Python 中等
标签：面向对象, 多继承, MRO, C3线性化

📑 题目

解释Python多继承的MRO（方法解析顺序）及其C3线性化算法。

💡 参考答案

MRO（Method Resolution Order）是Python确定多继承中方法调用顺序的机制。当子类调用某个方法时，Python按照MRO顺序在基类链中查找。

Python 3使用C3线性化算法计算MRO，保证：
1. 子类在父类之前。
2. 如果类继承自多个父类，按声明顺序从左到右查找。
3. 保持一致的单调性：子类的MRO中父类的顺序与父类自身的MRO一致。

查看MRO：
- ClassName.__mro__：返回元组。
- ClassName.mro()：返回列表。

经典问题（菱形继承）：
- Python 2使用深度优先（旧式类），可能导致非预期的父类方法被调用。
- Python 3统一使用C3算法（新式类），解决了菱形继承问题。

super()函数：在多继承中，super()按照MRO顺序调用下一个类的方法，不是简单的调用父类方法。这是协作式多重继承的基础。

注意事项：
- 多继承虽然强大，但会增加代码复杂度，推荐优先使用组合（composition）。
- Mixin模式是多继承的推荐用法：将功能拆分为小的Mixin类。

💻 代码示例

class A: def greet(self): print('A') class B(A): def greet(self): print('B') class C(A): def greet(self): print('C') class D(B, C): pass print(D.__mro__) # (<class 'D'>, <class 'B'>, <class 'C'>, <class 'A'>, <class 'object'>) D().greet() # B（先查找B，再C，再A） # super()在多继承中的行为 class Base: def __init__(self): print('Base.__init__') super().__init__() class Mixin1(Base): def __init__(self): print('Mixin1.__init__') super().__init__() class Mixin2(Base): def __init__(self): print('Mixin2.__init__') super().__init__() class Child(Mixin1, Mixin2): pass Child() # Mixin1 -> Mixin2 -> Base
