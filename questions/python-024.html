asyncio异步编程

Python 中等
标签：并发, asyncio, 协程, 异步编程

📑 题目

解释Python asyncio异步编程的核心概念、使用方式及适用场景。

💡 参考答案

asyncio是Python的异步I/O框架，基于事件循环和协程实现，适合处理大量I/O密集型任务。

核心概念：
- 协程（coroutine）：使用async def定义的函数，调用后返回协程对象。
- 事件循环（event loop）：异步程序的核心调度器，负责执行协程和处理I/O事件。
- await：暂停协程执行，等待异步操作完成，期间事件循环可以执行其他协程。
- Task：对协程的封装，提供并发执行的能力。
- Future：代表未来某个时刻的结果。

关键API：
- asyncio.run()：运行最高层级的入口点。
- asyncio.create_task()：将协程包装为Task并发执行。
- asyncio.gather()：并发运行多个协程，等待全部完成。
- asyncio.wait()：更灵活的并发控制（FIRST_COMPLETED等）。
- async with / async for：异步上下文管理和异步迭代。

适用场景：
- 高并发网络请求（HTTP客户端、WebSocket）。
- 数据库异步操作。
- 高性能Web服务器（FastAPI、Sanic）。
- 实时通信和流式处理。

注意事项：
- asyncio是单线程的，CPU密集型任务会阻塞事件循环。
- 不能在同步函数中直接调用异步函数，需要使用asyncio.run()或loop.run_until_complete()。
- 避免在协程中调用阻塞的同步I/O。

💻 代码示例

import asyncio async def fetch_data(url, delay): print(f'开始获取 {url}') await asyncio.sleep(delay) # 模拟异步I/O print(f'完成获取 {url}') return f'{url} 的数据' async def main(): # 并发执行多个协程 results = await asyncio.gather( fetch_data('url1', 2), fetch_data('url2', 1), fetch_data('url3', 3), ) print(results) # 使用Task task1 = asyncio.create_task(fetch_data('url4', 1)) task2 = asyncio.create_task(fetch_data('url5', 2)) r1 = await task1 r2 = await task2 asyncio.run(main()) # 总耗时约3秒（最慢的任务时间），而非6秒
