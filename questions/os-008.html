用户态与内核态

操作系统 中等
标签：用户态, 内核态, 系统调用, 特权级

📑 题目

用户态和内核态有什么区别？如何切换？

💡 参考答案

用户态：
- 普通应用程序运行的状态
- 权限受限，不能直接访问硬件和内核数据
- x86中为特权级3（Ring 3）

内核态：
- 操作系统内核运行的状态
- 拥有最高权限，可以访问所有硬件和内存
- x86中为特权级0（Ring 0）

切换方式（用户态→内核态）：
1. 系统调用（syscall）：应用程序主动请求OS服务
2. 中断（Interrupt）：硬件发出的异步通知
3. 异常（Exception）：程序错误（除零、缺页等）

切换过程：
1. 保存用户态上下文（寄存器、程序计数器等）
2. 切换到内核栈
3. 设置特权级为内核态
4. 执行内核代码
5. 返回时恢复用户态上下文

💻 代码示例

// Linux系统调用示例 #include <unistd.h> #include <sys/syscall.h> // 使用glibc封装 int fd = open("file.txt", O_RDONLY); read(fd, buffer, 1024); close(fd); // 直接使用syscall（底层） // syscall(SYS_open, "file.txt", O_RDONLY); // syscall(SYS_read, fd, buffer, 1024); // 常见系统调用 // 文件：open/read/write/close // 进程：fork/exec/wait/exit // 内存：brk/mmap/munmap // 通信：pipe/shmget/socket
