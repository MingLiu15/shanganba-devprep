C++20 Concepts

C/C++ 困难
标签：Concepts, C++20, 模板约束, 概念, 类型约束

📑 题目

什么是C++20 Concepts？它解决了什么问题？

💡 参考答案

Concepts：模板参数的约束条件，编译期类型检查。

解决的问题：
1. 模板错误信息晦涩难懂（SFINAE的替代）
2. 模板参数要求不明确
3. 编译期类型检查

标准Concepts：
- std::integral：整型
- std::floating_point：浮点型
- std::copyable：可复制
- std::default_initializable：可默认构造
- std::random_access_iterator：随机访问迭代器

自定义Concepts：
- 使用requires表达式定义约束
- 可以组合（&&, ||）

优势：
- 错误信息清晰
- 代码可读性高
- 支持重载解析

💻 代码示例

#include <concepts> // 使用标准Concept template<std::integral T> T add(T a, T b) { return a + b; } // 自定义Concept template<typename T> concept Addable = requires(T a, T b) { { a + b } -> std::convertible_to<T>; }; template<typename T> concept HasSize = requires(T t) { { t.size() } -> std::integral; }; // 组合Concepts template<typename T> concept Container = HasSize<T> && requires(T t) { { t.begin() } -> std::input_or_output_iterator; { t.end() } -> std::input_or_output_iterator; }; // 使用Concept template<Container C> void printSize(const C& container) { std::cout << container.size(); } // requires子句 template<typename T> requires std::copyable<T> && std::equality_comparable<T> void process(T value) { // ... }
