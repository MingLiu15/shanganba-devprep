读者-写者问题

操作系统 中等
标签：读者写者, 同步, 读写锁, 并发

📑 题目

什么是读者-写者问题？如何实现？

💡 参考答案

问题描述：
- 多个读者可以同时读
- 写者必须独占访问（读写互斥、写写互斥）

两种策略：
1. 读者优先：
- 只要有读者在读，新读者可以直接进入
- 写者可能饥饿（一直有新读者到来）

2. 写者优先：
- 有写者等待时，新读者必须等待
- 读者可能饥饿

实现要点：
- 互斥锁：保护共享变量
- 读计数器：记录当前读者数量
- 信号量：控制写者访问

实际应用：
- 数据库读写锁
- 配置文件读写
- 缓存系统

💻 代码示例

// 读者优先实现 #include <pthread.h> int read_count = 0; pthread_mutex_t count_mutex = PTHREAD_MUTEX_INITIALIZER; pthread_mutex_t rw_mutex = PTHREAD_MUTEX_INITIALIZER; void reader() { pthread_mutex_lock(&count_mutex); read_count++; if (read_count == 1) pthread_mutex_lock(&rw_mutex); // 第一个读者加锁 pthread_mutex_unlock(&count_mutex); // 读取数据... pthread_mutex_lock(&count_mutex); read_count--; if (read_count == 0) pthread_mutex_unlock(&rw_mutex); // 最后一个读者解锁 pthread_mutex_unlock(&count_mutex); } void writer() { pthread_mutex_lock(&rw_mutex); // 写者独占 // 写入数据... pthread_mutex_unlock(&rw_mutex); }
