死锁的产生条件与避免

操作系统 中等
标签：死锁, 互斥, 循环等待, 银行家算法

📑 题目

死锁产生的必要条件是什么？如何预防和避免？

💡 参考答案

死锁四个必要条件（同时满足）：
1. 互斥：资源一次只能被一个进程使用
2. 占有且等待：持有资源同时等待其他资源
3. 不可抢占：资源不能被强制剥夺
4. 循环等待：存在进程等待环路

预防（破坏必要条件）：
1. 破坏互斥：某些资源可以共享（如只读文件）
2. 破坏占有等待：一次性申请所有资源
3. 破坏不可抢占：允许抢占资源
4. 破坏循环等待：按顺序编号，按序申请

避免（运行时判断）：
- 银行家算法：在分配前检查分配后是否安全
- 需要知道每个进程的最大资源需求

检测与恢复：
- 资源分配图检测环路
- 恢复：终止进程、抢占资源

💻 代码示例

// 银行家算法核心逻辑 bool isSafe(int available[], int max[][N], int alloc[][N], int need[][N]) { bool finish[N] = {false}; int work[N]; memcpy(work, available, sizeof(work)); int count = 0; while (count < N) { bool found = false; for (int i = 0; i < N; i++) { if (!finish[i] && canSatisfy(need[i], work)) { for (int j = 0; j < N; j++) work[j] += alloc[i][j]; finish[i] = true; count++; found = true; } } if (!found) return false; // 不安全 } return true; // 安全
