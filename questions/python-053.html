Python项目打包与发布

Python 中等
标签：setuptools, pyproject.toml, pip, wheel, 打包发布

📑 题目

如何打包和发布Python项目？pyproject.toml和setup.py的区别？

💡 参考答案

**现代打包方式（PEP 517/518）：**
- pyproject.toml：统一项目配置文件
- 替代setup.py、setup.cfg、requirements.txt

**核心概念：**
1. **wheel（.whl）**：二进制分发格式，安装快
2. **sdist（.tar.gz）**：源码分发格式
3. **build**：构建工具（pip install build）

**pyproject.toml结构：**
- [build-system]：构建后端（setuptools/hatch/poetry）
- [project]：项目元数据（PEP 621）
- [tool.xxx]：工具配置

**发布流程：**
1. 编写pyproject.toml
2. pip install build
3. python -m build
4. twine upload dist/*

**依赖管理：**
- dependencies：运行时依赖
- optional-dependencies：可选依赖
- dev-dependencies：开发依赖

💻 代码示例

# ===== pyproject.toml 示例 ===== config = """ [build-system] requires = ["setuptools>=68.0", "wheel"] build-backend = "setuptools.build_meta" [project] name = "my-package" version = "1.0.0" description = "My awesome package" readme = "README.md" license = {text = "MIT"} requires-python = ">=3.8" authors = [{name = "Author", email = "author@example.com"}] # 运行时依赖 dependencies = [ "requests>=2.28.0", "numpy>=1.20.0", ] # 可选依赖 [project.optional-dependencies] dev = ["pytest>=7.0", "black", "mypy"] docs = ["sphinx", "sphinx-rtd-theme"] # 入口点 [project.scripts] mytool = "my_package.cli:main" [project.entry-points."myapp.plugins"] plugin1 = "my_package.plugin1" # setuptools配置 [tool.setuptools.packages.find] where = ["src"] """ print(config) # ===== 构建和发布命令 ===== build_commands = """ # 1. 安装构建工具 pip install build twine # 2. 构建包（生成dist/目录下的.whl和.tar.gz） python -m build # 3. 检查包 twine check dist/* # 4. 上传到PyPI twine upload dist/* # 5. 安装测试 pip install my-package # 从PyPI安装 pip install ./dist/my_package-1.0.0-py3-none-any.whl # 从本地安装 # ===== setup.py（旧方式，仍广泛使用）===== # from setuptools import setup, find_packages # setup( # name="my-package", # version="1.0.0", # packages=find_packages(where="src"), # install_requires=["requests>=2.28.0"], # entry_points={"console_scripts": ["mytool=my_package.cli:main"]}, # ) """ print(build_commands)
