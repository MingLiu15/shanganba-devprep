pytest进阶：标记与过滤

自动化测试 中等
标签：pytest, mark, 标记, 过滤, 分组测试

📑 题目

pytest的mark标记如何使用？如何过滤测试？

💡 参考答案

**内置标记：**
- `@pytest.mark.skip`：跳过测试
- `@pytest.mark.skipif`：条件跳过
- `@pytest.mark.xfail`：预期失败
- `@pytest.mark.parametrize`：参数化

**自定义标记：**
在pytest.ini或pyproject.toml中注册

**运行过滤：**
- `-m MARK`：运行指定标记的测试
- `-k EXPRESSION`：按名称匹配
- `--co`：只收集不运行
- `-v`：详细输出
- `-x`：第一个失败即停止

💻 代码示例

import pytest import sys # pytest.ini 配置 # [pytest] # markers = # slow: marks tests as slow # smoke: smoke tests # integration: integration tests # 自定义标记 @pytest.mark.slow def test_slow_operation(): """耗时测试""" import time time.sleep(5) assert True @pytest.mark.smoke def test_smoke(): """冒烟测试""" assert True @pytest.mark.integration def test_integration(): """集成测试""" assert True # 组合标记 @pytest.mark.smoke @pytest.mark.slow def test_smoke_slow(): assert True # 条件跳过 @pytest.mark.skipif(sys.platform == "win32", reason="不支持Windows") def test_linux_only(): assert True # 预期失败 @pytest.mark.xfail(reason="已知bug") def test_known_bug(): assert False # 参数化 + 标记 @pytest.mark.parametrize("input,expected", [ (1, 2), (2, 4), pytest.param(3, 6, marks=pytest.mark.smoke), ]) def test_double(input, expected): assert input * 2 == expected # ===== 运行命令 ===== # 运行slow标记的测试 # pytest -m slow # 运行smoke标记但不是slow的测试 # pytest -m "smoke and not slow" # 按名称过滤 # pytest -k "test_smoke" # 排除某些测试 # pytest -m "not slow" # 列出所有标记 # pytest --markers
