死锁的产生条件和避免方法

C/C++ 中等
标签：死锁, 互斥锁, 同步, 多线程

📑 题目

什么是死锁？如何避免？

💡 参考答案

死锁四个必要条件（同时满足）：
1. 互斥：资源独占
2. 占有且等待：持有资源同时等待其他资源
3. 不可抢占：资源只能主动释放
4. 循环等待：形成等待环路

避免方法：
1. 锁顺序一致：所有线程按相同顺序获取锁
2. 使用std::lock同时获取多个锁
3. 使用try_lock，失败则释放已持有锁
4. 使用超时锁
5. 避免在持有锁时调用外部代码
6. 使用更高层次的同步原语（如线程安全队列）

💻 代码示例

std::mutex m1, m2; // 危险：不同顺序导致死锁 void thread1() { lock_guard<mutex> lock1(m1); // ... lock_guard<mutex> lock2(m2); // 可能死锁 } void thread2() { lock_guard<mutex> lock1(m2); // 顺序相反！ lock_guard<mutex> lock2(m1); } // 安全：统一顺序 void safe_thread() { lock(m1, m2); // 同时获取，避免死锁 lock_guard<mutex> lock1(m1, adopt_lock); lock_guard<mutex> lock2(m2, adopt_lock); }
