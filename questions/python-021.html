dict的底层实现（哈希表）

Python 中等
标签：数据结构, 字典, 哈希表, 底层实现

📑 题目

解释Python字典的底层实现原理，包括哈希表结构、冲突解决和扩容机制。

💡 参考答案

Python字典基于哈希表实现，Python 3.6+保持插入顺序。

底层结构：
- 哈希表由两个数组组成：indices数组（存储索引）和entries数组（存储键值对）。
- 每个entry包含：hash值、key指针、value指针。
- indices数组用于快速定位，entries数组按插入顺序存储，保证有序性。

哈希冲突解决：
- 使用开放寻址法（Open Addressing），不是链地址法。
- 发生冲突时，通过探测序列（线性探测）寻找下一个空位。
- 探测公式：i = (5*i + 1) mod 2^n，使用二次探测减少聚集。

哈希计算：
- 调用对象的__hash__()方法获取哈希值。
- 哈希值与表大小取模确定初始位置。
- 发生冲突时按探测序列查找。

扩容机制：
- 维护负载因子（元素数/表大小），当负载因子超过2/3时触发扩容。
- 扩容时创建更大的哈希表（通常为原来的2-3倍），重新哈希所有元素。
- 扩容操作是O(n)时间复杂度，但均摊后每次操作仍为O(1)。

时间复杂度：
- 平均查找、插入、删除：O(1)。
- 最坏情况（大量冲突）：O(n)。

key的要求：必须是可哈希的（不可变类型），实现了__hash__和__eq__。

# 哈希冲突演示 # 自定义类作为dict的key class Point: def __init__(self, x, y): self.x, self.y = x, y def __hash__(self): return hash((self.x, self.y)) def __eq__(self, other): return self.x == other.x and self.y == other.y d = {Point(1,2): 'A', Point(3,4): 'B'} print(d[Point(1,2)]) # A # 字典性能特点 import timeit print(timeit.timeit('d[999999]', setup='d={i:i for i in range(1000000)}')) # 查找接近O(1) # 不可哈希类型 # {[1,2]: 'value'} # TypeError: unhashable type: 'list' # Python 3.7+ 字典保持插入顺序 d = {'a': 1, 'b': 2, 'c': 3} print(list(d.keys())) # ['a', 'b', 'c']
