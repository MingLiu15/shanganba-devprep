GIL全局解释器锁

Python 中等
标签：并发, GIL, 多线程, 性能

📑 题目

解释Python GIL（全局解释器锁）的原理、影响及应对方案。

💡 参考答案

GIL（Global Interpreter Lock）是CPython实现中的一个互斥锁，确保同一时刻只有一个线程执行Python字节码。

存在原因：
- CPython的内存管理（引用计数）不是线程安全的，GIL防止多个线程同时修改引用计数导致内存问题。
- 简化了CPython的实现，使大量C扩展无需考虑线程安全。

影响：
- CPU密集型任务无法利用多核：多线程在CPython中无法真正并行执行。
- I/O密集型任务影响较小：I/O操作时会释放GIL，其他线程可以执行。

应对方案：
- CPU密集型任务使用multiprocessing模块，每个进程有独立的GIL和Python解释器。
- 使用C扩展（如NumPy）在执行计算时释放GIL。
- 使用concurrent.futures的ProcessPoolExecutor。
- 考虑使用不依赖GIL的实现（如PyPy的某些模式、Jython、IronPython）。
- Python 3.13+引入了实验性的free-threaded模式（PEP 703），可以禁用GIL。

注意：GIL只存在于CPython中，不意味着Python语言本身有这个限制。

💻 代码示例

import threading import multiprocessing import time # GIL对CPU密集型任务的影响 def cpu_bound(n): count = 0 for i in range(n): count += i return count # 多线程（受GIL限制） start = time.time() threads = [threading.Thread(target=cpu_bound, args=(10**7,)) for _ in range(4)] for t in threads: t.start() for t in threads: t.join() print(f'多线程: {time.time()-start:.2f}s') # 并不比单线程快 # 多进程（绕过GIL） start = time.time() pool = multiprocessing.Pool(4) pool.map(cpu_bound, [10**7]*4) pool.close() print(f'多进程: {time.time()-start:.2f}s') # 接近4倍加速
