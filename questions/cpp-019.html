迭代器失效问题

C/C++ 中等
标签：迭代器, 失效, vector, list, map

📑 题目

哪些操作会导致迭代器失效？如何避免？

💡 参考答案

vector/string：
- 插入/删除：可能导致所有迭代器失效（扩容时）
- push_back/emplace_back：可能导致所有迭代器失效

deque：
- 首尾以外的插入/删除：所有迭代器失效
- 首尾操作：只有被删除元素的迭代器失效

list/forward_list：
- 只有被删除元素的迭代器失效

map/set/unordered_map：
- 只有被删除元素的迭代器失效

避免方法：
1. 操作后重新获取迭代器
2. 使用返回的新迭代器
3. 先存储需要删除的key，再统一删除

💻 代码示例

vector<int> v = {1, 2, 3, 4, 5}; auto it = v.begin() + 2; v.push_back(6); // 可能导致it失效 // 安全做法：操作后重新获取 it = v.begin() + 2; // 删除时更新迭代器 for (auto it = v.begin(); it != v.end(); ) { if (*it % 2 == 0) it = v.erase(it); // erase返回下一个有效迭代器 else ++it; }
