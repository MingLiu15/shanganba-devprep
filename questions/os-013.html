共享内存的原理

操作系统 中等
标签：共享内存, mmap, IPC, 同步

📑 题目

共享内存是如何实现的？为什么是最快的IPC方式？

💡 参考答案

原理：
- 将同一块物理内存映射到多个进程的虚拟地址空间
- 进程直接读写内存，无需内核介入数据传输

实现方式：
1. System V共享内存：shmget/shmat/shmdt/shmctl
2. POSIX共享内存：shm_open/mmap/munmap/shm_unlink
3. mmap映射文件：将文件映射到内存

为什么最快：
- 其他IPC（管道、消息队列）需要数据在用户空间和内核空间之间拷贝
- 共享内存直接在用户空间读写，零拷贝
- 仅在建立映射时需要内核介入

必须配合同步机制：
- 信号量
- 互斥锁
- 因为没有内核参与的传输，无法自动同步

💻 代码示例

// System V共享内存 #include <sys/shm.h> #include <sys/sem.h> int shmid = shmget(IPC_PRIVATE, 1024, IPC_CREAT|0666); char* addr = (char*)shmat(shmid, NULL, 0); // 使用信号量同步 int semid = semget(IPC_PRIVATE, 1, IPC_CREAT|0666); // P操作（等待） struct sembuf p = {0, -1, 0}; semop(semid, &p, 1); // 读写共享内存 strcpy(addr, "data from process A"); // V操作（释放） struct sembuf v = {0, 1, 0}; semop(semid, &v, 1); shmdt(addr); // 取消映射 shmctl(shmid, IPC_RMID, NULL); // 删除
