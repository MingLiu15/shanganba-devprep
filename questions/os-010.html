硬链接与软链接的区别

操作系统 简单
标签：硬链接, 软链接, 符号链接, inode, Linux

📑 题目

硬链接和软链接（符号链接）有什么区别？

💡 参考答案

硬链接（Hard Link）：
- 指向同一个inode的不同文件名
- 删除原文件不影响硬链接（inode引用计数减1，为0时才真正删除）
- 不能跨文件系统
- 不能链接目录
- 创建：ln source target

软链接/符号链接（Symbolic Link）：
- 是一个独立的文件，存储目标文件的路径
- 类似于Windows的快捷方式
- 删除原文件后软链接失效（悬空链接）
- 可以跨文件系统
- 可以链接目录
- 创建：ln -s source target

区别总结：
1. inode：硬链接共享inode，软链接有独立inode
2. 删除原文件：硬链接不受影响，软链接失效
3. 跨文件系统：硬链接不行，软链接可以
4. 链接目录：硬链接不行，软链接可以

💻 代码示例

// 创建链接 // 硬链接 // $ ln file.txt hardlink.txt // 软链接 // $ ln -s file.txt softlink.txt // 查看inode // $ ls -li // 1234567 -rw-r--r-- 2 user file.txt // 1234567 -rw-r--r-- 2 user hardlink.txt // 同一inode // 7654321 lrwxrwxrwx 1 user softlink.txt -> file.txt // 查看链接数 // $ stat file.txt // Link: 2 (硬链接数) // C语言创建链接 #include <unistd.h> link("file.txt", "hardlink.txt"); // 硬链接 symlink("file.txt", "softlink.txt"); // 软链接
