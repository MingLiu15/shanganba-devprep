Linux Capabilities

操作系统 困难
标签：Capabilities, 权限, 安全, 容器, 特权

📑 题目

什么是Linux Capabilities？它如何提升系统安全性？

💡 参考答案

Capabilities是Linux对传统root权限的细粒度拆分，将超级用户权限分解为多个独立的权限单元。

传统问题：
- root用户拥有所有权限，风险集中
- 普通程序需要root权限时，获得全部权限

Capabilities解决方式：
- 将root权限拆分为多个capability
- 程序只需所需的最小权限

常见Capabilities：
- CAP_CHOWN：修改文件所有者
- CAP_KILL：发送信号给任意进程
- CAP_NET_BIND_SERVICE：绑定特权端口（<1024）
- CAP_NET_ADMIN：网络管理操作
- CAP_SYS_ADMIN：系统管理（相当于大部分root权限）
- CAP_SYS_PTRACE：ptrace系统调用
- CAP_SETUID：修改进程UID

容器安全应用：
- 容器内进程默认丢弃大部分capabilities
- 只保留必要的capabilities（如CAP_NET_BIND_SERVICE）
- 即使容器内是root，权限也受限

查看和设置：
- capsh：查看当前capabilities
- setcap：给可执行文件添加capabilities
- Docker --cap-add/--cap-drop

💻 代码示例

// 查看进程的capabilities // $ cat /proc/<pid>/status | grep Cap // CapInh: 继承的capabilities // CapPrm: 允许的capabilities // CapEff: 有效的capabilities // CapBnd: 边界capabilities // 给程序添加capabilities // $ sudo setcap cap_net_bind_service=+ep /path/to/program // Docker使用capabilities // $ docker run --cap-drop=ALL --cap-add=NET_BIND_SERVICE nginx // C语言操作capabilities #include <sys/capability.h> // 获取当前capabilities cap_t caps = cap_get_proc(); cap_value_t cap_list[1] = {CAP_NET_BIND_SERVICE}; cap_set_flag(caps, CAP_EFFECTIVE, 1, cap_list, CAP_SET); cap_set_proc(caps); // 查看文件capabilities // $ getcap /usr/bin/ping // /usr/bin/ping = cap_net_raw+ep
