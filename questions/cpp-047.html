拷贝省略RVO与NRVO

C/C++ 困难
标签：RVO, NRVO, 拷贝省略, 编译器优化, 返回值优化

📑 题目

什么是RVO和NRVO？它们如何优化性能？

💡 参考答案

RVO（Return Value Optimization，返回值优化）：
- 函数返回临时对象时，编译器直接在调用者的内存中构造对象
- 省略了拷贝/移动构造函数的调用
- C++17中RVO是强制性的（保证省略）

NRVO（Named Return Value Optimization，命名返回值优化）：
- 函数返回命名局部变量时，编译器优化掉拷贝
- C++17中NRVO不强制，但大多数编译器支持

触发条件：
- 返回与函数返回类型相同的对象
- 不能有多个返回路径返回不同对象
- 不能返回不同作用域的局部变量

影响：
- 即使有拷贝/移动构造函数，也可能被省略
- C++17起，即使省略了拷贝/移动，拷贝/移动构造函数也必须可访问
- 语义上仍调用构造函数（析构函数也会调用）

💻 代码示例

// RVO示例 std::vector<int> createVector() { return std::vector<int>{1, 2, 3}; // RVO：直接在调用者处构造，无拷贝 } auto v = createVector(); // 无拷贝/移动 // NRVO示例 std::string buildString() { std::string result; result += "hello"; result += " world"; return result; // NRVO：省略拷贝 } // RVO失效的情况 std::string broken(int flag) { std::string a = "a"; std::string b = "b"; if (flag) return a; // 两个不同对象 return b; // NRVO可能失效 }
