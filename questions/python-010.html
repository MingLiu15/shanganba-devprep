Python多线程与多进程

Python 中等
标签：并发, 多线程, 多进程, GIL

📑 题目

比较Python中多线程和多进程的区别、适用场景及实现方式。

💡 参考答案

多线程（threading）：
- 共享同一进程的内存空间，线程间通信方便。
- 受GIL限制，CPU密集型任务无法真正并行。
- 适合I/O密集型任务（网络请求、文件读写、数据库操作）。
- 线程创建开销小，切换速度快。
- 需要注意线程安全问题（使用Lock、RLock、Semaphore等同步原语）。

多进程（multiprocessing）：
- 每个进程有独立的内存空间和GIL，可真正利用多核。
- 适合CPU密集型任务（数值计算、图像处理、机器学习）。
- 进程创建开销大，内存占用高。
- 进程间通信需要使用Queue、Pipe、Value、Array或Manager。
- 不存在GIL限制，可以充分利用多核CPU。

选择建议：
- I/O密集型 -> 多线程或asyncio。
- CPU密集型 -> 多进程。
- 混合型 -> 多进程 + 每个进程内多线程/协程。
- 推荐使用concurrent.futures的高级接口ThreadPoolExecutor和ProcessPoolExecutor。

💻 代码示例

import threading import multiprocessing from concurrent.futures import ThreadPoolExecutor, ProcessPoolExecutor import time # 多线程 - I/O密集型 def io_task(url): time.sleep(1) # 模拟I/O return url with ThreadPoolExecutor(max_workers=5) as pool: results = list(pool.map(io_task, ['url1','url2','url3'])) # 多进程 - CPU密集型 def cpu_task(n): return sum(i*i for i in range(n)) with ProcessPoolExecutor(max_workers=4) as pool: results = list(pool.map(cpu_task, [10**6]*4)) # 线程安全 - Lock lock = threading.Lock() counter = 0 def increment(): global counter for _ in range(100000): with lock: counter += 1
