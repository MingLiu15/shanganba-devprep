extern "C"的作用

C/C++ 中等
标签：extern C, name mangling, C兼容

📑 题目

为什么要使用extern "C"？

💡 参考答案

原因：C++支持函数重载，编译器会对函数名进行修饰（name mangling），生成唯一的符号名。C语言不支持重载，不进行修饰。

extern "C"作用：
1. 告诉C++编译器，这部分按C方式编译，不进行name mangling
2. 使C++代码可以调用C库
3. 使C代码可以调用C++库（需要C++端用extern "C"导出）

使用场景：
- 调用C标准库
- 调用系统API
- 编写跨语言库

💻 代码示例

// C++调用C代码 extern "C" { #include <cstdio> // 或声明C函数 int c_function(int x); } // C++代码导出给C使用 extern "C" int cpp_function(int x) { // 实现 return x * 2; } // 条件编译，C和C++都能包含 #ifdef __cplusplus extern "C" { #endif void my_function(); #ifdef __cplusplus } #endif
