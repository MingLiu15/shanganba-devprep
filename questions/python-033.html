Python的MRO算法（C3线性化）

Python 困难
标签：高级特性, MRO, C3线性化, 多继承

📑 题目

详细解释Python的C3线性化算法及其在MRO计算中的应用。

💡 参考答案

C3线性化是Python 3中计算多继承方法解析顺序（MRO）的算法，由一系列线性化约束产生。

C3算法规则：
1. 子类优先于父类。
2. 父类按声明顺序从左到右排列。
3. 如果一个类出现在多个父类的继承链中，只保留第一次出现的位置（单调性）。

算法步骤：
- MRO的计算可以看作合并多个线性化列表的过程。
- 每一步选择一个合法的类（不在任何列表的尾部）加入结果。
- 如果没有合法的类可选，说明继承结构不合法，抛出TypeError。

具体计算过程（以类D(B, C)为例）：
1. L[D] = D + merge(L[B], L[C], [B, C])
2. L[B] = B + merge(L[A], [A]) = [B, A, object]
3. L[C] = C + merge(L[A], [A]) = [C, A, object]
4. merge([B, A, object], [C, A, object], [B, C])
5. 选B（不在任何尾部），然后选C，然后选A，最后选object
6. L[D] = [D, B, C, A, object]

无效继承示例：
- class X(A, B): pass 和 class Y(B, A): pass，如果Z(X, Y)会导致TypeError，因为A和B的相对顺序矛盾。

super()与MRO的关系：
- super()不是简单的调用父类方法，而是按照MRO顺序调用下一个类的方法。
- super(ClassName, self)从MRO中ClassName之后开始查找。
- 这是协作式多重继承的基础。

💻 代码示例

# C3线性化计算示例 class O: pass class A(O): pass class B(O): pass class C(O): pass class D(B, C): pass class E(C, A): pass class F(D, E): pass print(F.__mro__) # (F, D, B, E, C, A, O, <class 'object'>) # 手动计算 L[F]: # L[O] = [O] # L[A] = [A, O] # L[B] = [B, O] # L[C] = [C, O] # L[D] = [D] + merge(L[B], L[C], [B, C]) # = [D, B, C, O] # L[E] = [E] + merge(L[C], L[A], [C, A]) # = [E, C, A, O] # L[F] = [F] + merge(L[D], L[E], [D, E]) # = [F, D, B, E, C, A, O] # 无效继承 # class X(A, B): pass # class Y(B, A): pass # class Z(X, Y): pass # TypeError: Cannot create a consistent MRO
