property装饰器与描述符

Python 中等
标签：面向对象, property, 描述符, 封装

📑 题目

解释Python中property装饰器的用法及描述符协议的原理。

💡 参考答案

property装饰器：将方法转换为属性访问，实现getter/setter/deleter，在不改变调用方式的前提下添加验证逻辑。

property用法：
- @property定义getter方法。
- @attr.setter定义setter方法。
- @attr.deleter定义deleter方法。
- 优势：保持接口一致性，可以在访问/修改属性时添加逻辑。

描述符协议：
- 实现了__get__、__set__、__delete__中至少一个方法的类就是描述符。
- 数据描述符：同时实现__get__和__set__，优先级高于实例属性。
- 非数据描述符：只实现__get__，优先级低于实例属性。
- 描述符是property、classmethod、staticmethod、__slots__的底层实现机制。

描述符的查找顺序（优先级从高到低）：
1. 数据描述符（类属性中定义的描述符）。
2. 实例__dict__。
3. 非数据描述符。

property本质上就是一个数据描述符类。

💻 代码示例

# property装饰器 class Student: def __init__(self, name, score): self.name = name self._score = score @property def score(self): return self._score @score.setter def score(self, value): if not 0 <= value <= 100: raise ValueError('分数必须在0-100之间') self._score = value s = Student('Tom', 85) print(s.score) # 85（通过getter） s.score = 95 # 通过setter验证 # s.score = 200 # ValueError # 自定义描述符 class Validated: def __init__(self, min_val, max_val): self.min_val, self.max_val = min_val, max_val def __set_name__(self, owner, name): self.name = '_' + name def __get__(self, obj, objtype=None): return getattr(obj, self.name, None) def __set__(self, obj, value): if not self.min_val <= value <= self.max_val: raise ValueError(f'{self.name} 超出范围') setattr(obj, self.name, value)
