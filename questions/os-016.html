中断与异常

操作系统 中等
标签：中断, 异常, 陷阱, 故障, 终止

📑 题目

中断和异常有什么区别？各有哪些类型？

💡 参考答案

中断（Interrupt）：
- 来自CPU外部的异步事件
- 硬件中断：I/O完成、时钟、键盘等
- 可屏蔽中断（INTR）：可通过中断控制器屏蔽
- 不可屏蔽中断（NMI）：紧急硬件故障

异常（Exception）：
- CPU执行指令时产生的同步事件
- 分为三类：
1. 故障（Fault）：可恢复，重新执行指令
- 缺页中断
2. 陷阱（Trap）：有意为之，执行后继续下一条
- 系统调用（int 0x80 / syscall）
3. 终止（Abort）：不可恢复，终止程序
- 硬件错误、非法指令

中断处理流程：
1. 保存现场（寄存器、PC等）
2. 确定中断源（查询中断向量表）
3. 执行中断处理程序
4. 恢复现场，返回被中断的程序

💻 代码示例

// 中断向量表（x86 IDT） struct IDTEntry { uint16_t offset_low; // 处理函数地址低16位 uint16_t selector; // 代码段选择子 uint8_t reserved; uint8_t type_attr; // 类型属性 uint16_t offset_mid; // 中间16位 uint32_t offset_high; // 高32位 uint32_t reserved2; }; // 常见中断/异常向量号 // 0: 除零错误（Fault） // 13: GPF通用保护（Fault） // 14: 缺页中断（Fault） // 0x80: 系统调用（Trap） // 0x21-0x2F: 硬件中断
