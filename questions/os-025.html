DMA（直接内存访问）

操作系统 中等
标签：DMA, IO, 中断, 性能优化

📑 题目

什么是DMA？它有什么作用？

💡 参考答案

DMA（Direct Memory Access）：
- 允许外设直接与内存交换数据，不经过CPU
- 由DMA控制器（DMAC）完成数据传输

工作流程：
1. CPU向DMA控制器发送传输请求
2. DMA控制器向总线仲裁器申请总线控制权
3. 获得总线控制权后，DMA直接读写内存
4. 传输完成后，DMA向CPU发送中断通知

优点：
1. 解放CPU：CPU在DMA传输期间可以执行其他任务
2. 速度快：DMA是硬件传输，比CPU逐字节传输快
3. 减少中断次数：整块数据传输完才中断一次

应用场景：
- 磁盘读写
- 网络数据收发
- 显存数据传输
- 音频数据播放

与中断驱动IO对比：
- 中断驱动：每个字节/字都中断CPU，开销大
- DMA：整块数据传输完才中断，开销小

💻 代码示例

// DMA传输流程 // 1. CPU配置DMA控制器 // - 源地址（外设寄存器） // - 目标地址（内存） // - 传输字节数 // - 传输方向（读/写） // 2. DMA控制器接管总线 // - CPU释放总线控制权 // - DMA直接在设备和内存间传输数据 // 3. 传输完成 // - DMA释放总线 // - 向CPU发送中断 // Linux查看DMA // $ dmesg | grep -i dma // $ cat /proc/dma
