📑 题目

什么是RCU？它适用于什么场景？

💡 参考答案

RCU（Read-Copy-Update）：读多写少的同步机制，读者无锁，写者复制更新。

核心思想：
1. 读者：直接读取，不加锁（极快）
2. 写者：复制数据，修改副本，原子替换指针
3. 旧数据：延迟释放，等待所有读者退出

适用场景：
- 读多写少（读者占99%+）
- Linux内核：路由表、文件系统、进程链表
- 需要极致读性能的场景

三个关键操作：
1. rcu_read_lock/unlock：读者临界区标记
2. synchronize_rcu：等待所有读者退出（宽限期）
3. call_rcu：注册回调，宽限期后释放旧数据

限制：
- 只保护指针，不保护数据
- 写者开销大（复制+延迟释放）
- 不能阻塞在RCU临界区内

💻 代码示例

// RCU简化概念 // 实际RCU是Linux内核机制，用户空间可用类似思想 #include <atomic> #include <memory> template<typename T> class RCUPtr { std::atomic<T*> ptr; std::atomic<int> readers{0}; public: T* read_lock() { readers++; return ptr.load(); } void read_unlock() { readers--; } void update(T* newPtr) { T* old = ptr.exchange(newPtr); // 等待所有读者退出 while (readers > 0) { std::this_thread::yield(); } delete old; // 安全删除 } }; // Linux内核RCU使用示例（概念） // struct my_struct *ptr; // // // 读者 // rcu_read_lock(); // p = rcu_dereference(ptr); // // 使用p... // rcu_read_unlock(); // // // 写者 // new_p = kmalloc(...); // *new_p = *p; // new_p->field = new_value; // rcu_assign_pointer(ptr, new_p); // synchronize_rcu(); // 等待宽限期 // kfree(p);
