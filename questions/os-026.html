交换空间与内存扩展

操作系统 简单
标签：Swap, 虚拟内存, 内存管理, OOM

📑 题目

什么是交换空间（Swap）？操作系统如何管理内存不足的情况？

💡 参考答案

交换空间（Swap）：
- 磁盘上用于扩展物理内存的区域
- Linux中为swap分区或swap文件
- 将不活跃的内存页换出到磁盘，需要时换入

页面置换时机：
1. 物理内存不足时，内核换出冷门页面
2. 通过LRU等算法选择被换出的页面

OOM（Out of Memory）：
- 物理内存和交换空间都耗尽
- Linux的OOM Killer会选择进程杀死
- 选择策略：分数越高越容易被杀（考虑内存占用、优先级等）

Linux内存管理：
- swappiness：控制换出倾向（0-100）
- overcommit_memory：是否允许超量分配
- cgroups：限制容器/进程的内存使用

查看内存和swap：
- free -h：查看内存和swap使用
- /proc/meminfo：详细内存信息
- swapon -s：查看swap设备

💻 代码示例

// 查看内存使用 // $ free -h // total used free shared buff/cache available // Mem: 7.8G 3.2G 1.5G 256M 3.0G 4.1G // Swap: 2.0G 128M 1.9G // 查看swap // $ swapon --show // 创建swap文件 // $ fallocate -l 2G /swapfile // $ chmod 600 /swapfile // $ mkswap /swapfile // $ swapon /swapfile // 调整swappiness（降低换出倾向） // $ sysctl vm.swappiness=10 // 查看OOM分数 // $ cat /proc/<pid>/oom_score
