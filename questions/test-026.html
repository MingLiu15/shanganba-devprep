测试驱动开发（TDD）

自动化测试 中等
标签：TDD, 测试驱动开发, 红绿重构, 单元测试

📑 题目

什么是TDD？TDD的工作流程是什么？

💡 参考答案

**TDD（测试驱动开发）：**
先写测试，再写代码，通过测试来驱动设计。

**红绿重构循环：**
1. **红**：写一个失败的测试
2. **绿**：写最少代码让测试通过
3. **重构**：优化代码，保持测试通过

**TDD优势：**
- 高测试覆盖率
- 更好的设计（松耦合）
- 活文档（测试即文档）
- 更安全地重构

**TDD挑战：**
- 学习曲线陡峭
- 初期开发速度较慢
- 不适合探索性开发
- UI/UX开发较难TDD

💻 代码示例

# TDD示例：实现一个计算器 # 步骤1：写测试（红） # test_calculator.py def test_add(): calc = Calculator() assert calc.add(2, 3) == 5 def test_subtract(): calc = Calculator() assert calc.subtract(5, 3) == 2 # 步骤2：实现代码（绿） # calculator.py class Calculator: def add(self, a, b): return a + b def subtract(self, a, b): return a - b # 步骤3：重构 # 优化代码结构，保持测试通过 class Calculator: def add(self, a, b): """Add two numbers.""" return a + b def subtract(self, a, b): """Subtract b from a.""" return a - b def multiply(self, a, b): """Multiply two numbers.""" return a * b def divide(self, a, b): """Divide a by b.""" if b == 0: raise ValueError("Cannot divide by zero") return a / b
