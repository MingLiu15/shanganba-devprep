内存屏障与内存序

C/C++ 困难
标签：内存屏障, memory_order, 原子操作, 无锁编程

📑 题目

什么是内存屏障？C++中的memory_order有哪些？

💡 参考答案

内存屏障（Memory Barrier）：防止编译器和CPU对内存操作进行重排序的指令。

为什么需要：
- 编译器优化可能重排指令
- CPU乱序执行
- 多核CPU缓存不一致

C++11 memory_order：
1. memory_order_relaxed：最宽松，只保证原子性
2. memory_order_consume：数据依赖顺序（很少使用）
3. memory_order_acquire：读操作，之后的读写不会重排到前面
4. memory_order_release：写操作，之前的读写不会重排到后面
5. memory_order_acq_rel：读写操作，acquire+release
6. memory_order_seq_cst：顺序一致性，最严格（默认）

使用原则：
- 默认用seq_cst，简单但性能较低
- 性能敏感场景用acquire/release
- relaxed只用于计数器等简单场景

💻 代码示例

#include <atomic> std::atomic<int> data(0); std::atomic<bool> ready(false); // 生产者 void producer() { data.store(42, std::memory_order_relaxed); ready.store(true, std::memory_order_release); // 写屏障 } // 消费者 void consumer() { while (!ready.load(std::memory_order_acquire)) // 读屏障 ; // 保证看到data=42 int value = data.load(std::memory_order_relaxed); } // 无锁计数器用relaxed class LockFreeCounter { std::atomic<int> count{0}; public: void increment() { count.fetch_add(1, std::memory_order_relaxed); } int get() { return count.load(std::memory_order_relaxed); } };
