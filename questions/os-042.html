fork的写时复制COW

操作系统 中等
标签：fork, COW, 写时复制, 进程创建, 内存优化

📑 题目

解释fork的写时复制(Copy-On-Write)机制。

💡 参考答案

写时复制(COW)：fork创建子进程时，不立即复制父进程内存，而是共享同一物理内存，直到某一方修改时才复制。

工作流程：
1. fork时：父子进程页表指向相同物理页，标记为只读
2. 读操作：直接共享，无需复制
3. 写操作：触发缺页中断，复制该页，然后修改

优点：
1. 加速fork：无需复制整个地址空间
2. 节省内存：共享只读数据
3. 减少IO：不触及磁盘

应用场景：
- 进程创建
- 内存映射文件
- 虚拟化（KVM/QEMU）
- 容器镜像层

注意：
- 大量写操作会导致频繁复制，反而降低性能
- 与exec配合时，COW避免了不必要的复制

💻 代码示例

// COW示意（概念性） // fork前：父进程有数据页 // fork后：父子共享同一物理页（只读标记） #include <unistd.h> #include <sys/wait.h> int main() { int data = 100; // 父子进程共享此页 pid_t pid = fork(); if (pid == 0) { // 子进程：此时data与父进程共享物理内存 printf("Child before modify: %d\n", data); // 写操作触发COW，复制该页 data = 200; printf("Child after modify: %d\n", data); } else { wait(NULL); // 父进程的data不受影响 printf("Parent: %d\n", data); // 输出100 } } // 查看COW统计（Linux） // $ grep cow /proc/vmstat // pswpin - 页面换入 // pswpout - 页面换出 // mmap的MAP_PRIVATE也使用COW void* addr = mmap(NULL, size, PROT_READ|PROT_WRITE, MAP_PRIVATE, fd, 0); // 对addr的写操作会触发COW
