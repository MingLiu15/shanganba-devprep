C++异常处理机制

C/C++ 中等
标签：异常, try, catch, throw, 异常安全

📑 题目

解释C++异常处理机制及异常安全级别。

💡 参考答案

异常处理三要素：
- throw：抛出异常对象
- try：标记可能抛出异常的代码块
- catch：捕获并处理异常

异常传播：异常沿调用栈向上查找匹配的catch，栈上的局部对象自动析构（栈展开）。

标准异常类层次：
exception → runtime_error → range_error, overflow_error
exception → logic_error → invalid_argument, out_of_range

异常安全级别（从弱到强）：
1. 无保证：可能泄漏资源
2. 基本保证：无资源泄漏，对象处于有效但未知状态
3. 强异常保证：操作失败时状态回滚
4. 不抛出保证：noexcept，保证不抛异常

最佳实践：
- 析构函数不要抛异常
- 构造函数可以抛异常
- 优先使用RAII而非手动try/catch
- 按引用捕获：catch(const std::exception& e)

💻 代码示例

try { throw std::runtime_error("出错了"); } catch (const std::runtime_error& e) { std::cerr << e.what() << std::endl; } catch (const std::exception& e) { std::cerr << "异常: " << e.what() << std::endl; } catch (...) { // 捕获所有异常 std::cerr << "未知异常" << std::endl; throw; // 重新抛出 } // noexcept保证 void safeFunc() noexcept { // 保证不抛异常 }
