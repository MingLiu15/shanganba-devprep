pytest进阶：conftest与Fixture共享

自动化测试 中等
标签：pytest, conftest, Fixture, 共享配置

📑 题目

pytest的conftest.py如何使用？Fixture如何共享？

💡 参考答案

**conftest.py作用：**
1. 定义共享的fixture
2. 定义共享的hooks
3. 自动被同目录及子目录的测试发现

**Fixture作用域：**
- `function`：每个测试函数调用一次（默认）
- `class`：每个测试类调用一次
- `module`：每个模块调用一次
- `package`：每个包调用一次
- `session`：整个测试会话调用一次

**Fixture特点：**
- 支持依赖注入
- 支持参数化
- 支持yield（setup/teardown）
- 支持autouse自动使用

💻 代码示例

# conftest.py - 共享fixture配置 import pytest import tempfile import os from selenium import webdriver # 数据库fixture（session级别） @pytest.fixture(scope="session") def db_connection(): """创建数据库连接，整个测试会话共享""" conn = create_db_connection("test.db") yield conn conn.close() # 临时文件fixture（function级别） @pytest.fixture def temp_file(): """创建临时文件，每个测试独立""" fd, path = tempfile.mkstemp() yield path os.close(fd) os.unlink(path) # 测试数据fixture @pytest.fixture def sample_user(): return {"username": "testuser", "email": "test@example.com"} # 参数化fixture @pytest.fixture(params=["chrome", "firefox"]) def browser(request): """多浏览器测试""" if request.param == "chrome": driver = webdriver.Chrome() else: driver = webdriver.Firefox() yield driver driver.quit() # 自动使用的fixture @pytest.fixture(autouse=True) def setup_test_env(): """每个测试前自动设置环境""" os.environ["TEST_MODE"] = "true" yield del os.environ["TEST_MODE"] # ===== 测试文件中使用 ===== # test_user.py def test_with_db(db_connection): """使用共享数据库连接""" cursor = db_connection.cursor() cursor.execute("SELECT * FROM users") assert cursor.fetchall() is not None def test_with_temp_file(temp_file): """使用临时文件""" with open(temp_file, 'w') as f: f.write("test content") assert os.path.exists(temp_file) def test_with_sample(sample_user): """使用测试数据""" assert sample_user["username"] == "testuser" def test_multiple_browsers(browser): """多浏览器测试""" browser.get("https://example.com") assert browser.title
