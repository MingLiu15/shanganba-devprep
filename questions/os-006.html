页面置换算法

操作系统 中等
标签：页面置换, LRU, FIFO, OPT, 缺页中断

📑 题目

常见的页面置换算法有哪些？

💡 参考答案

1. OPT（最优置换）：
- 淘汰将来最长时间不使用的页面
- 理论最优，无法实现，用于比较基准

2. FIFO（先进先出）：
- 淘汰最早进入内存的页面
- 实现简单，但可能产生Belady异常（增加物理帧反而缺页增多）

3. LRU（最近最少使用）：
- 淘汰最近最长时间未使用的页面
- 性能接近OPT，但实现开销大
- 可用计数器或栈实现

4. LFU（最不经常使用）：
- 淘汰访问次数最少的页面
- 可能忽略最近访问情况

5. Clock（时钟算法）：
- LRU的近似，使用访问位和修改位
- 页面组织为循环链表，指针扫描
- 实现简单，性能较好

实际系统中常用LRU近似算法（如Clock的改进版）。

💻 代码示例

// LRU页面置换（双向链表+哈希表） class LRUCache { list<int> lru_list; // 最近使用的在前面 unordered_map<int, list<int>::iterator> map; int capacity; public: void access(int page) { if (map.count(page)) { lru_list.erase(map[page]); // 移到前面 } else if (lru_list.size() >= capacity) { int victim = lru_list.back(); lru_list.pop_back(); map.erase(victim); // 淘汰 } lru_list.push_front(page); map[page] = lru_list.begin(); } };
