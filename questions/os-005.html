虚拟内存的工作原理

操作系统 中等
标签：虚拟内存, 页表, MMU, 地址空间

📑 题目

什么是虚拟内存？它是如何工作的？

💡 参考答案

虚拟内存：为每个进程提供独立的、连续的虚拟地址空间，通过页表映射到物理内存。

工作原理：
1. CPU发出虚拟地址
2. MMU（内存管理单元）通过页表将虚拟地址转换为物理地址
3. 如果页面在物理内存中，直接访问
4. 如果不在（缺页），从磁盘加载到物理内存

页表结构：
- 多级页表：节省内存（如4级页表）
- TLB（快表）：缓存最近使用的页表项，加速地址转换

页面置换算法：
- FIFO：先进先出，可能产生Belady异常
- LRU：最近最少使用，性能好但开销大
- LFU：最不经常使用
- Clock：LRU的近似实现

优点：
- 内存隔离，进程间互不影响
- 可以使用比物理内存更大的地址空间
- 实现内存共享（共享库）

💻 代码示例

// 虚拟地址结构（假设4KB页面，32位地址） // [虚拟页号(20位)] [页内偏移(12位)] // 页表项结构 struct PageTableEntry { unsigned int frame_num : 20; // 物理页框号 unsigned int present : 1; // 是否在内存 unsigned int modified : 1; // 是否修改（脏页） unsigned int referenced: 1; // 是否被访问 unsigned int protection: 3; // 读写执行权限 }; // 地址转换 uint32_t translate(PageTableEntry* pageTable, uint32_t virtualAddr) { int page = virtualAddr >> 12; // 高20位 int offset = virtualAddr & 0xFFF; // 低12位 if (pageTable[page].present) return (pageTable[page].frame_num << 12) | offset; else trigger_page_fault(page); // 缺页中断 }
