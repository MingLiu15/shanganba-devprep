内存映射mmap

操作系统 中等
标签：mmap, 内存映射, 文件IO, 共享内存, 零拷贝

📑 题目

什么是mmap？它有什么优势和应用场景？

💡 参考答案

mmap（Memory Map）：将文件或其他对象映射到进程的虚拟地址空间，实现文件IO的内存化访问。

工作原理：
1. 建立文件到内存的映射关系
2. 访问内存即读写文件（由OS自动同步）
3. 支持私有映射（写时复制）和共享映射

优势：
1. 减少数据拷贝：用户空间直接访问文件，无需read/write系统调用
2. 减少内存占用：多个进程共享同一物理页
3. 随机访问：可直接访问文件任意位置
4. 简化编程：像访问内存一样访问文件

应用场景：
1. 大文件处理（数据库、日志）
2. 进程间共享内存
3. 动态链接库加载
4. 零拷贝传输（配合sendfile）

注意事项：
- 文件大小变化可能导致SIGBUS
- 需要手动同步（msync）保证数据落盘
- 不适合小文件（开销大）

💻 代码示例

#include <sys/mman.h> #include <sys/stat.h> #include <fcntl.h> #include <unistd.h> // 读取文件 void readFile(const char* filename) { int fd = open(filename, O_RDONLY); struct stat sb; fstat(fd, &sb); // 映射文件到内存 char* addr = (char*)mmap(NULL, sb.st_size, PROT_READ, MAP_PRIVATE, fd, 0); // 像访问数组一样访问文件 for (size_t i = 0; i < sb.st_size; i++) { putchar(addr[i]); } munmap(addr, sb.st_size); close(fd); } // 进程间共享内存 void sharedMemory() { int fd = shm_open("/myshm", O_CREAT | O_RDWR, 0666); ftruncate(fd, 4096); char* addr = (char*)mmap(NULL, 4096, PROT_READ | PROT_WRITE, MAP_SHARED, fd, 0); // 进程A写入 strcpy(addr, "Hello from Process A"); // 进程B可以直接读取 // printf("%s\n", addr); munmap(addr, 4096); shm_unlink("/myshm"); } // 查看进程的内存映射 // $ cat /proc/<pid>/maps
