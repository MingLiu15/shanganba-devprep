Python描述符协议

Python 困难
标签：高级特性, 描述符, descriptor, 元编程

📑 题目

解释Python描述符协议的原理、分类及实现方式。

💡 参考答案

描述符（Descriptor）是Python中实现属性访问控制的底层机制，是property、classmethod、staticmethod的底层实现。

描述符协议：一个类如果实现了以下方法中的一个或多个，就是描述符。
- __get__(self, obj, objtype=None)：访问属性时调用。
- __set__(self, obj, value)：设置属性时调用。
- __delete__(self, obj)：删除属性时调用。
- __set_name__(self, owner, name)：在创建拥有类时自动调用（Python 3.6+）。

分类：
1. 数据描述符（Data Descriptor）：同时实现__get__和__set__（或__delete__）。优先级高于实例__dict__。
2. 非数据描述符（Non-Data Descriptor）：只实现__get__。优先级低于实例__dict__。

属性查找顺序（重要）：
1. 数据描述符（类属性中的描述符）。
2. 实例__dict__。
3. 非数据描述符。
4. __getattr__（兜底方法）。

property的本质：property就是一个数据描述符类，内部实现了__get__、__set__、__delete__。

应用场景：
- 类型验证：确保属性值符合类型要求。
- 延迟计算：属性在首次访问时才计算。
- 属性通知：属性变化时触发回调。
- ORM字段映射：类似Django的Field。

函数和方法也是描述符：普通函数是非数据描述符，实现__get__返回绑定方法。

💻 代码示例

# 类型验证描述符 class TypedField: def __init__(self, field_type, field_name=None): self.field_type = field_type self.field_name = field_name def __set_name__(self, owner, name): self.field_name = name def __get__(self, obj, objtype=None): if obj is None: return self return obj.__dict__.get(self.field_name) def __set__(self, obj, value): if not isinstance(value, self.field_type): raise TypeError(f'{self.field_name} 必须是 {self.field_type}') obj.__dict__[self.field_name] = value class Person: name = TypedField(str) age = TypedField(int) p = Person() p.name = 'Tom' # OK p.age = 25 # OK # p.age = '25' # TypeError: age 必须是 <class 'int'> # 延迟计算描述符 class LazyProperty: def __init__(self, func): self.func = func def __get__(self, obj, objtype=None): if obj is None: return self value = self.func(obj) setattr(obj, self.func.__name__, value) # 缓存到实例 return value
