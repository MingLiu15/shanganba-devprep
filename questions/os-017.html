僵尸进程与孤儿进程

操作系统 简单
标签：僵尸进程, 孤儿进程, wait, init

📑 题目

什么是僵尸进程和孤儿进程？如何处理？

💡 参考答案

僵尸进程（Zombie）：
- 子进程终止后，父进程未调用wait/waitpid回收其退出状态
- 子进程的PCB（进程控制块）仍保留在内核中
- 占用PID和少量内存，无法被kill
- 大量僵尸进程会耗尽PID资源

处理方法：
- 父进程调用wait()/waitpid()回收
- 杀死父进程，僵尸进程被init进程接管并回收

孤儿进程（Orphan）：
- 父进程先于子进程终止
- 子进程被init进程（PID=1）收养
- init进程会自动调用wait回收
- 孤儿进程无害

预防僵尸进程：
1. 父进程调用wait/waitpid
2. 使用信号处理：SIGCHLD信号处理中调用wait
3. 两次fork：创建子进程再创建孙子进程，子进程退出，孙子进程成为孤儿

💻 代码示例

#include <sys/wait.h> // 方法1：父进程wait pid_t pid = fork(); if (pid == 0) { exit(0); } else { wait(NULL); } // 回收子进程 // 方法2：SIGCHLD信号处理 void sigchld_handler(int sig) { while (waitpid(-1, NULL, WNOHANG) > 0); } signal(SIGCHLD, sigchld_handler); // 方法3：两次fork避免僵尸 if (fork() == 0) { if (fork() == 0) { // 孙子进程，成为孤儿被init收养 do_work(); exit(0); } exit(0); // 子进程立即退出 } wait(NULL); // 回收子进程
