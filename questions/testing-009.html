契约测试（Contract Testing）

自动化测试 中等
标签：契约测试, Pact, API契约, 微服务测试, 消费者驱动

📑 题目

什么是契约测试？为什么需要契约测试？

💡 参考答案

契约测试：验证服务间交互契约的测试方式，确保消费者和提供者的契约一致。

为什么需要：
1. 微服务架构中服务间依赖复杂
2. 传统集成测试慢且不稳定
3. 服务独立部署，需要提前验证兼容性

契约测试类型：
1. 消费者驱动契约（CDC）：消费者定义期望，提供者验证
2. 提供者驱动契约：提供者定义API，消费者遵守

常用工具：
- Pact：最流行的契约测试框架
- Spring Cloud Contract：Java生态
- Pact Broker：契约版本管理

💻 代码示例

# Pact契约测试示例 from pact import Consumer, Provider import requests pact = Consumer('UserService').has_pact_with(Provider('UserAPI')) @pact.given('a user exists').upon_receiving('a request for user').with_request( method='GET', path='/users/1' ).will_respond_with(status=200, body={ 'id': 1, 'name': 'Alice', 'email': 'alice@example.com' }) def test_get_user(): with pact: result = requests.get(pact.uri + '/users/1') assert result.json()['name'] == 'Alice' # 提供者验证 # pact-verifier --provider UserAPI --pact-url ./pacts/userservice-userapi.json # Pact Broker配置 # 发布契约: pact-broker publish ./pacts --consumer-app-version 1.0.0 # 验证契约: pact-verifier --provider App --broker-base-url http://pact-broker
