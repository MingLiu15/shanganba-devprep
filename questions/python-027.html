Python异常处理机制

Python 简单
标签：基础语法, 异常处理, 错误处理

📑 题目

解释Python异常处理机制，包括try/except/else/finally的用法及自定义异常。

💡 参考答案

Python异常处理机制允许程序在运行时检测和处理错误，避免程序崩溃。

基本语法：
- try：包含可能抛出异常的代码。
- except：捕获并处理特定类型的异常。
- else：没有异常时执行的代码（可选）。
- finally：无论是否发生异常都会执行的代码（可选），通常用于资源清理。

异常层次结构：
- BaseException -> Exception -> ValueError, TypeError, KeyError, IndexError等。
- 通常只捕获Exception及其子类，不要捕获BaseException（包括KeyboardInterrupt和SystemExit）。

常见异常类型：
- ValueError：值不合法。
- TypeError：类型不匹配。
- KeyError：字典键不存在。
- IndexError：索引越界。
- AttributeError：属性不存在。
- FileNotFoundError：文件不存在。

自定义异常：
- 继承Exception类。
- 可以添加自定义属性和方法。
- 通常以Error结尾命名。

最佳实践：
- 尽量捕获具体的异常类型，避免裸except。
- 使用raise ... from ...保留异常链。
- 不要用异常来控制正常流程。
- 在finally中释放资源，或使用上下文管理器。

💻 代码示例

# 基本异常处理 try: result = 10 / 0 except ZeroDivisionError as e: print(f'除零错误: {e}') except (TypeError, ValueError) as e: print(f'类型或值错误: {e}') else: print('没有异常') finally: print('总是执行') # 自定义异常 class ValidationError(Exception): def __init__(self, field, message): self.field = field self.message = message super().__init__(f'{field}: {message}') def validate_age(age): if not isinstance(age, int): raise TypeError('年龄必须是整数') if age < 0 or age > 150: raise ValidationError('age', f'年龄{age}无效') try: validate_age(200) except ValidationError as e: print(e.field, e.message) # age 年龄200无效 # 异常链 try: int('abc') except ValueError as e: raise RuntimeError('解析失败') from e
