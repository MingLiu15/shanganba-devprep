迭代器与可迭代对象的区别

Python 简单
标签：基础语法, 迭代器, 协议

📑 题目

解释Python中可迭代对象（Iterable）和迭代器（Iterator）的区别。

💡 参考答案

可迭代对象（Iterable）：实现了__iter__()方法的对象，可以返回一个迭代器。list、dict、str、set等都是可迭代对象。可以用for循环遍历。

迭代器（Iterator）：实现了__iter__()和__next__()方法的对象。__iter__()返回自身，__next__()返回下一个元素，没有更多元素时抛出StopIteration。

核心区别：
- 可迭代对象不一定是迭代器，但迭代器一定是可迭代对象。
- 可迭代对象可以通过iter()转换为迭代器。
- 迭代器是一次性的，遍历完后不能重新开始；可迭代对象可以多次遍历。
- for循环的底层机制：先调用iter()获取迭代器，然后不断调用next()获取元素。

判断方法：
- isinstance(obj, collections.abc.Iterable) 判断是否可迭代。
- isinstance(obj, collections.abc.Iterator) 判断是否是迭代器。

自定义迭代器需要实现__iter__和__next__方法，通常配合实例变量记录迭代状态。

💻 代码示例

from collections.abc import Iterable, Iterator # 可迭代对象 vs 迭代器 lst = [1, 2, 3] print(isinstance(lst, Iterable)) # True print(isinstance(lst, Iterator)) # False it = iter(lst) print(isinstance(it, Iterator)) # True print(next(it)) # 1 print(next(it)) # 2 # 自定义迭代器 class CountDown: def __init__(self, start): self.current = start def __iter__(self): return self def __next__(self): if self.current <= 0: raise StopIteration self.current -= 1 return self.current + 1 for i in CountDown(3): print(i, end=' ') # 3 2 1
