SFINAE与enable_if

C/C++ 困难
标签：SFINAE, enable_if, 模板元编程, 类型萃取

📑 题目

解释SFINAE原则和std::enable_if的作用。

💡 参考答案

SFINAE（Substitution Failure Is Not An Error）：
- 替换失败不是错误
- 当模板参数替换导致无效代码时，编译器不会报错，而是丢弃该模板
- 用于根据类型特性选择不同的模板实现

std::enable_if：
- 利用SFINAE实现条件模板
- enable_if<true, T>::type 是 T
- enable_if<false, T>::type 未定义，导致SFINAE

应用场景：
1. 根据类型特性重载函数
2. 模板约束（C++20 Concepts的底层实现）
3. 编译期类型选择

C++20替代方案：
- Concepts提供更清晰的语法，替代enable_if

💻 代码示例

#include <type_traits> // 只接受整数类型的函数 template<typename T> typename std::enable_if<std::is_integral<T>::value, T>::type process(T value) { return value * 2; } // 只接受浮点类型的函数 template<typename T> typename std::enable_if<std::is_floating_point<T>::value, T>::type process(T value) { return value * 2.0; } // C++14简化写法 template<typename T, typename = std::enable_if_t<std::is_pointer<T>::value>> void handlePointer(T ptr) { // 只接受指针 } // 使用 process(5); // OK，整数版本 process(3.14); // OK，浮点版本 // process("hi"); // 错误，无匹配版本 // 与C++20 Concepts对比 // template<std::integral T> // T process(T value) { return value * 2; }
