IO多路复用

操作系统 困难
标签：select, poll, epoll, IO多路复用, 高并发

📑 题目

什么是IO多路复用？select、poll、epoll的区别？

💡 参考答案

IO多路复用：一个线程同时监视多个文件描述符，等待一个或多个IO事件就绪。

适用场景：高并发连接，每个连接大部分时间空闲。

select：
- 使用fd_set位图，最大1024个fd
- 每次调用拷贝fd_set到内核
- 返回后遍历所有fd
- 时间复杂度O(n)

poll：
- 使用pollfd数组，无数量限制
- 仍需拷贝和遍历
- 时间复杂度O(n)

epoll（Linux最优）：
- epoll_create创建实例
- epoll_ctl添加/修改/删除fd（红黑树存储）
- epoll_wait只返回就绪的fd
- 时间复杂度O(就绪fd数)
- 支持LT（水平触发）和ET（边缘触发）
- 内核中维护就绪链表，无需每次遍历

💻 代码示例

// epoll服务端 int epfd = epoll_create1(0); struct epoll_event ev; ev.events = EPOLLIN; ev.data.fd = listen_fd; epoll_ctl(epfd, EPOLL_CTL_ADD, listen_fd, &ev); struct epoll_event events[1024]; while (true) { int n = epoll_wait(epfd, events, 1024, -1); for (int i = 0; i < n; i++) { if (events[i].data.fd == listen_fd) { int conn = accept(listen_fd, NULL, NULL); ev.events = EPOLLIN | EPOLLET; ev.data.fd = conn; epoll_ctl(epfd, EPOLL_CTL_ADD, conn, &ev); } else { handle(events[i].data.fd); } } }
