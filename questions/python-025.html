线程池与进程池（concurrent.futures）

Python 中等
标签：并发, 线程池, 进程池, concurrent.futures

📑 题目

介绍Python concurrent.futures模块中线程池和进程池的使用方法。

💡 参考答案

concurrent.futures提供了高级的异步执行接口，封装了threading和multiprocessing模块的复杂性。

ThreadPoolExecutor：
- 适用于I/O密集型任务。
- 线程复用，避免频繁创建销毁线程的开销。
- 通过max_workers参数控制最大线程数。

ProcessPoolExecutor：
- 适用于CPU密集型任务。
- 每个任务在独立进程中执行，绕过GIL限制。
- max_workers默认为CPU核心数。

核心方法：
- submit(fn, *args)：提交任务，返回Future对象。
- map(func, iterables)：类似内置map，但并发执行。
- shutdown(wait=True)：等待所有任务完成后关闭池（with语句自动调用）。

Future对象：
- result(timeout)：获取结果，可设置超时。
- done()：判断是否完成。
- add_done_callback(fn)：添加完成回调。
- cancel()：尝试取消任务。

异常处理：
- Future.result()会重新抛出任务中的异常。
- 可以使用concurrent.futures.as_completed()获取已完成的Future。

推荐使用with语句管理线程池/进程池，确保资源正确释放。

💻 代码示例

from concurrent.futures import ThreadPoolExecutor, ProcessPoolExecutor, as_completed import time # ThreadPoolExecutor - I/O密集型 def download(url): time.sleep(1) # 模拟网络请求 return f'{url} done' urls = [f'url_{i}' for i in range(5)] with ThreadPoolExecutor(max_workers=3) as executor: # 方式1：submit futures = {executor.submit(download, url): url for url in urls} for future in as_completed(futures): print(future.result()) # 方式2：map results = list(executor.map(download, urls)) print(results) # ProcessPoolExecutor - CPU密集型 def compute(n): return sum(i * i for i in range(n)) with ProcessPoolExecutor() as executor: future = executor.submit(compute, 10**7) print(future.result()) results = list(executor.map(compute, [10**6, 10**6, 10**6]))
