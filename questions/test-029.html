📑 题目

兼容性测试包含哪些类型？如何设计兼容性测试策略？

💡 参考答案

**兼容性测试类型：**

1. **浏览器兼容性：**
- Chrome、Firefox、Safari、Edge
- 不同版本（最新、上一个、稳定版）
- 内核差异（Blink、Gecko、WebKit）

2. **操作系统兼容性：**
- Windows、macOS、Linux
- 不同版本（Win10/11, macOS 12/13/14）

3. **设备兼容性：**
- 手机、平板、桌面
- 不同屏幕尺寸和分辨率
- 不同浏览器尺寸（响应式）

4. **数据库兼容性：**
- MySQL、PostgreSQL、MongoDB
- 不同版本

**测试策略：**
- 优先级矩阵（用户量 × 市场占有率）
- 云测试平台（BrowserStack、 Sauce Labs）
- 自动化截图对比
- 真机测试实验室

# 兼容性测试示例 # pytest参数化测试（浏览器） import pytest @pytest.mark.parametrize("browser", [ "chrome", "firefox", "safari", "edge" ]) def test_cross_browser(driver, browser): if browser == "chrome": options = webdriver.ChromeOptions() elif browser == "firefox": options = webdriver.FirefoxOptions() driver = webdriver.Remote(command_executor='http://selenium:4444', options=options) driver.get("https://example.com") assert driver.title == "Expected Title" # 响应式测试 @pytest.mark.parametrize("viewport", [ (1920, 1080, "desktop"), (768, 1024, "tablet"), (375, 667, "mobile") ]) def test_responsive_design(driver, viewport): width, height, device = viewport driver.set_window_size(width, height) driver.get("https://example.com/responsive") # 验证布局是否正确 navbar = driver.find_element(By.CLASS_NAME, "navbar") assert navbar.is_displayed() # 数据库兼容性测试 import pytest from sqlalchemy import create_engine @pytest.mark.parametrize("db", [ "mysql", "postgresql", "sqlite" ]) def test_database_operations(db): if db == "mysql": engine = create_engine("mysql+pymysql://user:pass@localhost/test") elif db == "postgresql": engine = create_engine("postgresql://user:pass@localhost/test") else: engine = create_engine("sqlite:///:memory:") # 测试CRUD操作 with engine.connect() as conn: conn.execute(text("SELECT 1"))
