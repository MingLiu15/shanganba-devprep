NUMA架构

操作系统 困难
标签：NUMA, 非统一内存访问, 多处理器, 内存架构

📑 题目

什么是NUMA架构？与SMP有什么区别？

💡 参考答案

NUMA（Non-Uniform Memory Access）：非统一内存访问架构。

特点：
- 每个CPU有本地内存，访问更快
- 访问其他CPU的内存需要通过互联（慢）
- 适合大规模多核系统（几十核以上）

与SMP对比：
| 特性 | SMP | NUMA |
|------|-----|------|
| 内存访问 | 统一延迟 | 本地快，远程慢 |
| 扩展性 | 差（<32核） | 好（数百核） |
| 复杂度 | 简单 | 复杂 |

NUMA优化：
1. 内存分配：在本地节点分配（numactl）
2. 线程绑定：将线程绑定到特定CPU
3. 数据局部性：减少跨节点访问

Linux支持：
- numactl：控制NUMA策略
- /proc/cpuinfo：查看NUMA信息
- 透明大页(THP)在NUMA上的优化

💻 代码示例

// 查看NUMA信息 // $ numactl --hardware // $ lscpu | grep NUMA // NUMA-aware内存分配 #include <numa.h> void* allocate_on_node(int node, size_t size) { return numa_alloc_onnode(size, node); } // 绑定线程到特定CPU #include <sched.h> void bind_to_cpu(int cpu) { cpu_set_t cpuset; CPU_ZERO(&cpuset); CPU_SET(cpu, &cpuset); sched_setaffinity(0, sizeof(cpuset), &cpuset); } // C++17 NUMA-aware allocator（概念） template<typename T> struct NumaAllocator { int node; T* allocate(size_t n) { return (T*)numa_alloc_onnode(n * sizeof(T), node); } void deallocate(T* p, size_t n) { numa_free(p, n * sizeof(T)); } }; // 使用 std::vector<int, NumaAllocator<int>> vec;
