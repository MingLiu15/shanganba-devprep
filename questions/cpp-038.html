GDB常用调试命令

C/C++ 中等
标签：GDB, 调试, 断点, Linux

📑 题目

列举GDB调试器的常用命令。

💡 参考答案

GDB是GNU调试器，用于调试C/C++程序。

启动：
- gdb ./program：启动调试
- gdb ./program core：调试core dump
- gdb attach PID：附加到运行中的进程

断点：
- break main/b main：在main函数设断点
- break 20：在第20行设断点
- break file.cpp:30：在指定文件行号设断点
- info breakpoints：查看断点
- delete 1：删除1号断点

执行：
- run/r：运行程序
- continue/c：继续执行
- next/n：单步执行（不进入函数）
- step/s：单步执行（进入函数）
- finish：运行到当前函数返回

查看：
- print/p var：打印变量值
- display var：每次停止时显示变量
- backtrace/bt：查看调用栈
- info locals：查看局部变量
- list/l：显示源代码

💻 代码示例

// 编译时加-g选项保留调试信息 g++ -g -o program program.cpp // GDB会话示例 $ gdb ./program (gdb) break main (gdb) run (gdb) next (gdb) print variable (gdb) continue (gdb) quit
