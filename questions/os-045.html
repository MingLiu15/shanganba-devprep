systemd工作原理

操作系统 中等
标签：systemd, init系统, 服务管理, Linux启动

📑 题目

systemd是什么？与传统init系统有什么区别？

💡 参考答案

systemd：Linux系统和服务管理器，PID 1进程，替代传统的SysV init。

核心特性：
1. 并行启动：依赖关系驱动，加快启动速度
2. 按需启动：socket激活、D-Bus激活
3. 自动依赖管理：服务间依赖自动处理
4. 日志统一：journald二进制日志
5. 快照与恢复：保存/恢复系统状态

与SysV init对比：
| 特性 | SysV init | systemd |
|------|-----------|---------|
| 启动方式 | 串行 | 并行 |
| 服务管理 | 脚本 | unit文件 |
| 日志 | 文本 | 二进制 |
| 依赖 | 手动 | 自动 |

常用命令：
- systemctl start/stop/restart/enable/disable service
- journalctl：查看日志
- systemd-analyze：分析启动时间

💻 代码示例

// systemd unit文件示例 /etc/systemd/system/myapp.service /* [Unit] Description=My Application After=network.target [Service] Type=simple ExecStart=/usr/bin/myapp Restart=on-failure User=myuser [Install] WantedBy=multi-user.target */ // 常用命令 // $ systemctl daemon-reload // $ systemctl start myapp // $ systemctl enable myapp // 开机启动 // $ systemctl status myapp // $ journalctl -u myapp -f // 查看日志 // 查看启动耗时 // $ systemd-analyze // $ systemd-analyze blame // $ systemd-analyze critical-chain // 创建定时任务（替代cron） // /etc/systemd/system/backup.timer /* [Unit] Description=Run backup daily [Timer] OnCalendar=daily Persistent=true [Install] WantedBy=timers.target */
