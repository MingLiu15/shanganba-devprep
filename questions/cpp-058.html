map和unordered_map的区别

C/C++ 中等
标签：map, unordered_map, 红黑树, 哈希表, STL

📑 题目

比较C++中map和unordered_map的区别，以及各自的适用场景。

💡 参考答案

map（红黑树实现）：
- 有序：按键的升序排列
- 查找/插入/删除：O(log n)
- 需要元素支持<比较运算符
- 迭代器遍历有序
- 内存开销较大（每个节点存储颜色、父子指针）

unordered_map（哈希表实现）：
- 无序：不保证元素顺序
- 查找/插入/删除：平均O(1)，最坏O(n)（哈希冲突时）
- 需要元素支持==比较和hash函数
- 内存开销较小（桶数组+链表）

选择原则：
- 需要有序遍历→map
- 只需要查找→unordered_map（更快）
- 需要范围查询（lower_bound/upper_bound）→map
- 键为自定义类型→map更容易（只需<），unordered_map需要自定义hash

注意：
- unordered_map在极端情况下（大量哈希冲突）性能退化
- map的性能稳定，不会退化

💻 代码示例

#include <map> #include <unordered_map> // map - 有序 std::map<std::string, int> m; m["banana"] = 3; m["apple"] = 5; m["cherry"] = 2; // 遍历顺序: apple, banana, cherry（按key排序） for (auto& [k, v] : m) { cout << k << ": " << v << endl; } // unordered_map - 无序 std::unordered_map<std::string, int> um; um["banana"] = 3; um["apple"] = 5; um["cherry"] = 2; // 遍历顺序不确定 // 性能对比 // 查找100万次: unordered_map约50ms, map约300ms // 插入100万次: unordered_map约80ms, map约400ms
