Python垃圾回收机制

Python 中等
标签：内存管理, 垃圾回收, 引用计数, 分代回收

📑 题目

详细说明Python的垃圾回收机制，包括引用计数、标记清除和分代回收。

💡 参考答案

Python使用三种机制协同进行垃圾回收：

1. 引用计数（主要机制）：
- 每个对象维护一个引用计数器，被引用时+1，引用解除时-1。
- 计数为0时立即回收内存。
- 优点：实时性好，回收及时。缺点：无法处理循环引用。

2. 标记-清除（处理循环引用）：
- 针对可能存在循环引用的容器对象（list、dict、class instance等）。
- 从根集合出发，标记所有可达对象，未被标记的对象视为垃圾。
- 会暂时将所有容器对象的引用计数复制一份，避免在标记过程中因计数变化导致误判。

3. 分代回收（优化性能）：
- 将对象分为三代（0、1、2），新对象在第0代。
- 每代存活过一次GC后晋升到下一代。
- GC频率随代数递减：第0代最频繁，第2代最少。
- 基于经验：大多数对象生命周期短，存活越久的对象越可能继续存活。

可以通过gc模块手动控制：gc.collect()强制回收，gc.set_threshold()设置阈值，gc.disable()禁用自动GC。

💻 代码示例

import gc import sys # 引用计数 a = [1, 2, 3] print(sys.getrefcount(a)) # 2 (a + getrefcount参数) b = a print(sys.getrefcount(a)) # 3 del b print(sys.getrefcount(a)) # 2 # 循环引用 x = [] y = [] x.append(y) y.append(x) del x, y # 引用计数不为0，但已无法访问 # 手动触发GC gc.collect() # 回收循环引用 # 查看GC信息 print(gc.get_threshold()) # (700, 10, 10) print(gc.get_count()) # 当前各代对象计数 # 禁用/启用自动GC gc.disable() gc.enable()
