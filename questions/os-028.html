Linux Cgroups（控制组）

操作系统 困难
标签：Cgroups, 资源限制, 容器, Linux内核, 资源管理

📑 题目

什么是Linux Cgroups？它如何限制资源使用？

💡 参考答案

Cgroups（Control Groups）是Linux内核提供的资源限制和管理机制，用于限制进程组的资源使用。

主要子系统（Controller）：
1. cpu：限制CPU使用率
- cpu.shares：CPU时间片分配权重
- cpu.cfs_quota_us/cpu.cfs_period_us：硬限制CPU使用率

2. memory：限制内存使用
- memory.limit_in_bytes：内存上限
- memory.swappiness：swap使用倾向

3. blkio：限制块设备IO
- blkio.throttle.read_bps_device：读速度限制
- blkio.throttle.write_bps_device：写速度限制

4. pids：限制进程数量
- pids.max：最大进程数

5. net_cls/net_prio：网络流量控制

6. devices：设备访问控制
- 允许/禁止访问特定设备

7. freezer：暂停/恢复进程组

使用方式：
- 手动操作/sys/fs/cgroup目录
- 使用systemd
- Docker等容器工具自动配置

💻 代码示例

// 手动创建cgroup并限制资源 // 1. 创建cgroup // $ sudo mkdir /sys/fs/cgroup/memory/mygroup // 2. 设置内存限制（100MB） // $ echo 104857600 | sudo tee /sys/fs/cgroup/memory/mygroup/memory.limit_in_bytes // 3. 将进程加入cgroup // $ echo <pid> | sudo tee /sys/fs/cgroup/memory/mygroup/cgroup.procs // C语言操作cgroup #include <stdio.h> void set_memory_limit(pid_t pid, long long limit) { FILE* f = fopen("/sys/fs/cgroup/memory/mygroup/cgroup.procs", "w"); fprintf(f, "%d", pid); fclose(f); f = fopen("/sys/fs/cgroup/memory/mygroup/memory.limit_in_bytes", "w"); fprintf(f, "%lld", limit); fclose(f); } // Docker默认cgroup路径 // /sys/fs/cgroup/memory/docker/<container_id>/
