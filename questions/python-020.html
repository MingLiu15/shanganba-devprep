Python单例模式的实现

Python 中等
标签：设计模式, 单例模式, 创建型模式

📑 题目

列举Python中实现单例模式的多种方式，并比较各自的优缺点。

💡 参考答案

单例模式确保一个类只有一个实例，并提供全局访问点。

常见实现方式：

1. 使用__new__方法：重写__new__控制实例创建，最常用的方式。简单直观，但__init__会被多次调用。

2. 使用装饰器：通过闭包保存唯一实例。灵活，可以给任意类添加单例行为，但不支持多线程。

3. 使用元类：通过元类控制类的实例化。最Pythonic的方式，但复杂度较高。

4. 使用模块：Python模块天然是单例的（模块只加载一次）。最简单推荐的方式。

5. 使用类变量：在类中保存实例引用。简单但不够优雅。

线程安全：
- 以上基本实现都不是线程安全的。
- 需要使用Lock或threading.RLock来保证线程安全。

优缺点比较：
- 模块方式最简单，推荐优先使用。
- __new__方式最常用，适合大多数场景。
- 装饰器方式灵活但可能影响isinstance检查。
- 元类方式最强大但复杂度最高。
