Seccomp（安全计算模式）

操作系统 困难
标签：Seccomp, 安全, 系统调用过滤, 容器, 沙箱

📑 题目

什么是Seccomp？它如何增强系统安全性？

💡 参考答案

Seccomp（Secure Computing Mode）是Linux内核的安全机制，用于限制进程可以使用的系统调用，实现系统调用级别的沙箱。

两种模式：
1. strict模式：
- 只允许read/write/exit/sigreturn四个系统调用
- 其他系统调用触发SIGKILL

2. filter模式（Seccomp-BPF）：
- 使用BPF（Berkeley Packet Filter）规则过滤系统调用
- 可以自定义允许/拒绝的系统调用
- 支持条件判断（如根据参数值过滤）

容器安全应用：
- Docker默认启用seccomp，禁用约44个危险系统调用
- 如mount、umount2、reboot等
- 可以自定义seccomp profile

常见被限制的系统调用：
- mount/umount：防止挂载恶意文件系统
- reboot：防止重启宿主机
- open_by_handle_at：防止容器逃逸
- ptrace：防止调试其他进程

绕过检测：
- 检查是否启用：prctl(PR_GET_SECCOMP)
- 默认模式返回2表示使用seccomp filter

💻 代码示例

// Seccomp-BPF示例 #include <seccomp.h> #include <sys/syscall.h> int main() { scmp_filter_ctx ctx = seccomp_init(SCMP_ACT_ALLOW); // 默认允许 // 拒绝mount系统调用 seccomp_rule_add(ctx, SCMP_ACT_KILL, SCMP_SYS(mount), 0); // 拒绝reboot seccomp_rule_add(ctx, SCMP_ACT_ERRNO(EPERM), SCMP_SYS(reboot), 0); // 只允许open读取（O_RDONLY） seccomp_rule_add(ctx, SCMP_ACT_ALLOW, SCMP_SYS(open), 1, SCMP_A1(SCMP_CMP_EQ, O_RDONLY)); seccomp_load(ctx); // 加载规则 // 此后进程受seccomp限制 mount(...); // 会被杀死 } // Docker使用自定义seccomp profile // $ docker run --security-opt seccomp=myprofile.json nginx // 查看进程的seccomp状态 // $ grep Seccomp /proc/<pid>/status // Seccomp: 2 (0:disabled, 1:strict, 2:filter)
