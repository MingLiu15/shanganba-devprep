用户线程与内核线程

操作系统 中等
标签：用户线程, 内核线程, 线程模型, 1:1, N:1, M:N

📑 题目

用户线程和内核线程有什么区别？线程模型有哪些？

💡 参考答案

用户线程：
- 在用户空间管理，OS不可见
- 切换不需要内核介入，速度快
- 一个线程阻塞可能导致整个进程阻塞
- 不能利用多核（取决于实现）

内核线程：
- 由OS内核管理
- 切换需要内核介入，开销大
- 一个线程阻塞不影响其他线程
- 可以利用多核并行

三种线程模型：
1. 一对一（1:1）：
- 每个用户线程对应一个内核线程
- Linux pthreads采用此模型
- 优点：真正并行，阻塞不影响其他线程
- 缺点：创建开销大，数量受限

2. 多对一（N:1）：
- 多个用户线程映射到一个内核线程
- 优点：创建切换快
- 缺点：不能并行，一个阻塞全部阻塞

3. 多对多（M:N）：
- M个用户线程映射到N个内核线程
- 综合以上优点
- 实现复杂（如Go的goroutine）

💻 代码示例

// Linux pthread（1:1模型） #include <pthread.h> void* thread_func(void* arg) { printf("线程ID: %lu\n", pthread_self()); return NULL; } int main() { pthread_t tid; pthread_create(&tid, NULL, thread_func, NULL); pthread_join(tid, NULL); } // Go goroutine（M:N模型） // go func() { // fmt.Println("goroutine") // }()
