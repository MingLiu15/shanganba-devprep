分页与分段

操作系统 中等
标签：分页, 分段, 虚拟内存, 内存管理

📑 题目

分页和分段有什么区别？

💡 参考答案

分页（Paging）：
- 将物理内存划分为固定大小的页框（通常4KB）
- 将逻辑地址空间划分为相同大小的页面
- 通过页表进行地址映射
- 优点：无外部碎片，内存利用率高
- 缺点：内部碎片（最后一页可能不满），页表占用内存

分段（Segmentation）：
- 按程序逻辑将地址空间划分为多个段（代码段、数据段、栈段）
- 每段有段号和段内偏移
- 通过段表进行地址映射
- 优点：符合程序逻辑，便于共享和保护
- 缺点：外部碎片（段间空闲区域）

段页式：
- 先分段再分页，结合两者优点
- 每段内部按页管理
- 现代操作系统常用方式

区别总结：
1. 分页大小固定，分段大小可变
2. 分页无外部碎片有内部碎片，分段相反
3. 分页对程序员透明，分段可见

💻 代码示例

// 分页地址结构（32位，4KB页面） // 虚拟地址: [页号(20位)] [页内偏移(12位)] // 物理地址: [页框号(20位)] [页内偏移(12位)] // 分段地址结构 // 虚拟地址: [段号(16位)] [段内偏移(16位)] // 段页式地址结构 // 虚拟地址: [段号] [页号] [页内偏移] // Linux查看页面大小 // $ getconf PAGE_SIZE // 4096 // 查看进程内存映射 // $ cat /proc/self/maps
